Presenter, Host and Participant

When you enter an HPE MyRoom event, you enter either as a presenter or a participant. Using a Presentation or Persistent key, you can enter the room either as a Host or a Participant. In this Help, views and limitations are given for Presenter, Host and Participant roles. Below is a table of the difference between these roles.

 

Presenter:   A user who entered a scheduled event using a key designed for presenters
  • Can import content, control participant features, run surveys, and save content such as Notes and Surveys.
  • Has access to all tools in an event.
  • Can grant participants access to People panel, Room chat, Private chat, Hand up, Applaud, Add questions, Add answers to questions, View all questions, File transfer, Microphone, Speaker, Video camera, Drawing tools, Handouts folder, Export content and Print content.
  • Can allow or disallow microphone and video camera individually for participants.
  • Can promote participants to presenter with reduced functionality and demote presenters to participants as well.
  • Can create Quick keys valid for 10 minutes.
Promoted to Presenter
  • If you are in a room when the preconference time for an event starts, you are automatically promoted to a presenter. This means that if you have Owner, Reader, Writer or Room Administrator roles or have used a Presentation Host or Persistent Host key to enter the room before the preconference time, you will be promoted to presenter, with full presenter capabilities, when the preconference time starts.
  • This is also the case if you have been dragged into a room by another presenter already in the event. You are dragged into an event as a presenter with full presenter capabilities.
Host:   A user who entered a room using a Presentation Host or Persistent Host key
  • Has full presenter capabilities.
  • If you are in a room as a Host when the preconference time for an event starts, you are automatically promoted to a presenter.
Participant:   A user who entered the room with a participant key.
  • Has limited permissions in the room. Presenters can give them permissions at any time during an event.
  • Can enter an event during the specified preconference time (usually 30 minutes) and during the scheduled event time.
  • According to rights granted by the Presenter(s)/Host(s), can see and use the People panel, Room chat, Private chat, Hand up, Applaud, Add questions, Add answers to questions, View all questions, File transfer, Microphone, Speaker, Video camera, Drawing tools, Erase annotations, Handouts folder, Export content and Print content.
  • Limitations of a participant promoted to presenter in an event are:
    • cannot create a Quick Key
    • cannot make other users presenters or participants
    • cannot eject others from room
  • If you are in a room using a Presentation or Persistent participant key when the preconference time for an event starts, you are not promoted to presenter.
