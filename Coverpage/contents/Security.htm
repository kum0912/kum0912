Security

It is important to protect data that users create and store from unauthorized use, disclosure, modification, or destruction. Ensuring that users have the appropriate authority to view the data, load new data, or update existing data is an important aspect of any product. The level of access to the data, the class of users, etc., is determined by the Admin User.

Permissions need to be assigned to different entities in modules and sub modules for a user to be able to perform specific activities such as:

Create, View or Delete Collections

Create, View or Delete Dataviews

Create, View or Delete Insights

View or Create a Scheduler Process

View or Create a Scheduler Program

Run Valuation

Simulate

Create Portfolio

Collections

Collections are building blocks of raw data. It is that set of data imported to the Analytics Application from an external application/domain/spreadsheet for analytical purposes. Importing data is a one-time activity performed at the time of configuration of the App. Based on the scheduler, the application refreshes the collection.

Assigning permissions to a role

To assign security permissions:

  • View: To grant read access,
  • Edit: To grant write access. User is granted permission to Edit as well as View,
  • Delete: To grant to delete. User is granted permission to Delete as well as Edit and View,
  • Edit Security: To grant Edit Security rights. The user is granted permission to Edit Security as well as view, edit and delete.

Select the Data Filter from the drop-down list, if the need is to add Data level security.

Data Filter

Select the Field (Dimension) and the Value for which the Data level security is applicable to the chosen role. This data level security will be honored for all the dataviews and insights that are linked to this Collection. The end User based on role and this data level security will see only the applicable records.

Click on +more link , to add more data filters as shown in the below image.

Make the desired changes and click on the Save icon to save the mapping.

To cancel the mapping, click on the Cancel icon .

To provide permission access for user roles, enable the check-box.

Dataviews

Dataview is a filtered and a formatted view of data. Dataviews provide a dynamic and interactive view of data available in the form of collections or datasets, thus will allow quicker analysis and prompt decision making. Dataviews offer easy at-a-glance accessibility to key filtered parameters and metrics. The intuitive visualizations aid in driving better decision making.

Users can create Dataviews by applying different aggregation parameters and visualize data in the form of charts. The Dataviews reflect change as the value of data changes.

The system provides users the ease of creating Dataviews while viewing, filtering, and selecting collections, selecting and formatting chart to designing the Dataviews, and visualizing the Dataviews, all-in-one screen without navigates to multiple screens.

Insights

Insights provide the user the ability to gain precise and deeper understanding of the data gleaned over with the use of Dataviews. The insights obtained through analytics are powerful in that it not only helps mitigate risk, detect and diagnose the cause for losses, but also helps predict and avoid damage thereby helping businesses gain profitably within shorter timelines.

Insights are the interpretations that the user can draw from the Dataviews. You can combine two or more Dataviews to form one Insight which gives a consolidated view of data.
