Summer school

 

The Summer school option provides Ontario summer schools and boards with the ability to generate the official Summer School Register. This register option is available to Ontario boards and schools set up with ‘school operating type’ of summer school.

 

This is a classroom-based register used to record enrolment and attendance for pupils enrolled in the following summer school courses or programs:

 

  • Credit courses delivered through classroom instruction
  • Credit courses delivered through cooperative education
  • Additional preparation for changing course types
  • Mathematics courses for changing course types
  • Credit recovery
  • Non-credit summer school programs for pupils with developmental disabilities

 

Summer school classes or courses:

 

  • May not begin until after the last school day in the school year, nor may they extend beyond the first school day of the following school year
  • Must be provided by the board between 8 AM and 5 PM

 

This register is for classroom instruction and cannot be used for pupils who are taking correspondence/self-study/e-learning courses through Continuing Education or summer day camps, sports camps, or daycare activies (with the exception of non-credit summer school programs for pupils with developmental disabilities).  

 

This register is used to record enrolment and attendance only for pupils:

 

  • Who were enrolled in a day school program during the school year offered by a board and has been admitted to a summer school program.
  • Who were enrolled in grades 1 to 12 in a private school and has been admitted to a Ministry-funded summer school program offered by a board.

 

Schools and boards should be familiar with the official Ministry enrolment register instructions for continuing education programs. These instructions are updated annually on the Ministry’s site http://www.edu.gov.on.ca/eng/policyfunding/forms.html. This includes detailed information on which courses to include on this register.

 

Please be familiar with the Enrolment > Enrolment register info option in the student’s record. This will provide detailed information on the enrolment register(s) the student will appear on, as well as high credit and regular FTE/ADE info. Hint: on the FTE and/or ADE card in the student’s record, there’s a Details section that may be expanded to show info on how FTE/ADE is calculated, and how the student’s data impacts the register. Summer school classes are reported as ADE.

 

Set all students Con. Ed. Student Type to ‘also attends day school’.

As part of the summer school setup, the school’s configuration can set a default Con. Ed. Student Type for students in the school.  This would typically be set to default the Con. Ed. Student Type of ‘Also Attends Day School (Full-time)”.

 

This report will only pick up students with the Con. Ed. Student Type set to:

 

  • Also Attends Day School (Full-Time)
  • Also Attends Day School (Part-Time)

 

Continuing education (CE) pupils will not appear on this register. Occasionally, a board will allow a CE student to register in a credit course at a summer school. These students should be in a separate class that will appear on the Register of Enrolment in Continuing Education Credit Courses.

 

Set up classes that need to appear on the Summer School Register

If a class is to appear in the Summer School Register, it must be set up (Setup > Class info > Classes) as follows:

 

  • In the summer school, the course offering type must be set to one of the following:

 

- Summer credit course

- Summer: Transfer credit in math

- Summer: Additional preparation for changing course types

- Program for developmentally disabled pupils*

 

  • Enter the most appropriate Delivery and Course delivery type

 

  • Since funding is based on attendance, make sure that the class schedule, start and end dates, and period times are accurate. Make sure each student’s add/drop dates are accurate (add date is the first day attending the class, drop date is the first date of non-attendance in the class).

 

* If the course offering type is ‘program for developmentally disabled pupils’, the course code will appear as DD on the summary page of the register, rather than the course code defined for the class.

 

Teachers track classroom attendance

Teachers are responsible for taking accurate classroom attendance, just like any regular class.  The class schedule, start/end dates, and the school’s calendar will define what dates the teacher is expected to take attendance.

 

Cancelled instructional sessions – Funded cancelled session “C”

Where an instructional session has been cancelled owing to exceptional circumstances (such as bad weather or an emergency) and the instructor is paid, the session will be recognized for funding purposes. These sessions are included in the total number of sessions recognized for funding. The funded cancelled session must be marked with a “C” on the register and the reason for the cancellation must be recorded. To cancel a funded session, follow these instructions:

 

  • Use Attendance > Enter attendance > Group Excursion.  Select the date and group of students (e.g. all students or a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option c Cancelled.

 

 

  • Record an attendance note describing the reason for the cancellation.
  • Apply (save) the attendance for the group.

 

Note: If the “C” session is rescheduled, the instructor should not be paid for the rescheduled session and the rescheduled session must not be included in the total number of sessions recognized for funding purposes. See Rescheduled instructional sessions.

 

Cancelled instructional sessions – Unfunded cancelled session “D”

Where an instructional session has been cancelled (I.e. the teacher is unavailable) and the instructor is not paid, the session will not be recognized for funding purposes. Do not include this session in the total number of sessions recognized for funding. The unfunded cancelled session must be marked with a “D” on the register and the reason for the cancellation must be recorded. To cancel an unfunded session, follow these instructions:

 

  • Use Attendance > Enter attendance > Group Excursion.  Select the date and group of students (e.g. a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option $ Cancelled Not Funded.

 

 

Note: If the “D” session is rescheduled, the instructor should be paid for the rescheduled session and the rescheduled session must be included in the total number of sessions recognized for funding purposes.

 

Rescheduled sessions  

If a new session is to be added to the class, typically the class end date is extended to accommodate the extra session. This is done using Setup > Class info > Classes.

 

If the rescheduled session is due to a session that is given the code “C” (funded cancelled session), the instructor should not be paid for the rescheduled session and the rescheduled session must not be included in the total number of sessions recognized for funding purposes. To reschedule a session, and have it not appear in the register, follow these instructions:

 

  • Use Attendance > Enter attendance > Group/excursion.  Select the date and group of students (e.g. All students or a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option ^ Non-instructional Day


  • Record an attendance note describing the reason the class is removed from the register (e.g. rescheduled session for the mm/yyyy funded cancelled session due to weather).
  • Apply (save) the attendance for the group.

 

Generate the Summer School Register

 

To access the register, select Ontario reporting > Enrolment registers > Summer school.

 

 

 

The form will look like this:  

 

 

The list of classes will default any classes that are set up to appear on this register. You may limit the classes using the filters, or select an individual or range of classes to print.  A separate register is generated for each selected class.

 

Select the Print button located at the bottom of the form. Once the report has been submitted to the task queue, you may leave this application or generate other reports. You will receive notification when your report is ready. Reports are saved in file manager under Provincial > Enrolment registers.

 

 

Hint: See previously created registers by selecting the folder icon to open file manager at this location.

 

The info generated on the register may also be exported in a CSV file by checking the option shown below. The CSV file will be generated along with the register report, as part of the submission to the task queue.   

 

 

Here is an example of the register report:

 

 

 

The top of the report will include the register name, school year, class (course code & section letter and course title).

 

 

For each pupil in the class, it will include the following info from the student’s record (in the Personal area):

 

 

The ‘Day school pupil’ column is populated with either FT (full-time), PT (part-time) based on the student’s field called ‘Con. Ed. Student Type’ (in student record under Enrolment > Enrolment register info). This report will only pick up students with the Con. Ed. Student Type set to:

 

  • Also Attends Day School (Full-Time)
  • Also Attends Day School (Part-Time)

 

However, if the course delivery type is ‘Program for developmentally disabled pupils’, the summer school is an elementary school, the student is included regardless of the Con. Ed. Student Type.

 

The ‘Residence Status’ column will show either POB (Pupil of the Board) or OP (Other Pupil). A student is considered an Other Pupil (OP) if their student board residence status does not use ministry code 01, 06, 07, or 09.

 

The ‘Day School Address’ column will use the summer student’s origin school as recorded in the student’s record in Enrolment > Student enrolment.

 

To the right of the student info it will show the student’s attendance at sessions. This info is based on the entry of attendance by the teachers. The dates are based on the setup of the class.

 

 

For each date, a blank will appear if the student is present.  If the student was not present at the session, one of the following 4 codes will appear:

 

A      Pupil absence from a session

—     Late admission or early withdrawal (based on the student’s class add/drop dates)

C       Funded cancelled session

D       Unfunded cancelled session

 

Each student’s total funded attendance sessions are summed in the far-right column.

   

 

 

If a student is an “Other Pupil”, their sessions will not be recognized for funding (their sessions are not summed). For pupils of the board, the funded session sum will include all sessions where the student was present (blank), Absent (A) *, or C (funded cancelled session).   

 

* if the student is absent 3 or more consecutive sessions, these absences are not funded (they are excluded from total). Cancelled sessions (“C” or “D”) do not ‘break’ the consecutive absent count.  For example, if a student had 5 consecutive absences in a row (July 4, 5, 6, 7, 10), a cancelled class on July 8 does not count as a consecutive absence, but it also does not break the count of consecutive absences.

 

The sessions recognized for funding will be summed for all students in the class, at the bottom of the register.

 

 

After the student details page, a summary page is included with an area for the principal to certify the register.

 

 

The ‘Hours per session’ is based on the daily average minutes of instruction set up for the class (using Setup > Class info > Classes). This may be based on period times, or it may be manually entered directly into the daily average minutes of instruction field. The minutes will be converted to hours on this register.

 

Funding - OnSIS reporting of classes on the Continuing Education Registers

Funding is based on the information submitted to OnSIS.  See OnSIS > Summer school > Summer school info for details.

 

From the Ministry’s instructions:

 

Elemental data is collected once a year on August 31 (July-August). Submit data through the Summer School submissions (School Special Condition Type “8 – Summer”).  

 

Course Offering Type to be reported for each course for each student. Course Offering Types include:  

 

  • Summer credit course  

  • Summer: Transfer credit on math  

  • Summer: Additional preparation for changing course types  

  • Elementary Program for pupils with developmental disabilities  

 

The following Class Enrolment fields are mandatory:  

 

  • Course Schedule Indicator  

  • Course Delivery  

  • Course Language of Instruction  

  • Course Code  

  • Course Start Date  

  • Course End Date  

  • Attempted Credit Value  

  • Earned Credit Value  

  • Final Mark  

 

The corresponding ADE value must also be reported for each course in the “ADE – Summer” field.

 

ADE calculations for classes on the Summer School registers

The calculation of the average daily enrolment (ADE) is calculated as follows:

ADE = # of sessions recognized for funding x hours per session / 950
