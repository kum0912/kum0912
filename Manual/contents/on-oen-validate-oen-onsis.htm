Validate OENs in OnSIS

Student OENs are the key identifier for students when submitting information to OnSIS. If OENs are manually entered there is always the chance for data entry errors. Ontario publicly funded schools and boards may have all OENs validated to ensure that they are accurate prior to submitting student data to OnSIS. To validate OENs in OnSIS, follow these steps.

1. Choose Ontario reporting, Ontario Education Numbers (OEN) from the main menu. Select the CREATE button on the Validate OENs in OnSIS card.

 

2. The Create batch file page will appear. The CREATE option simply requires you to select the CREATE BATCH FILE button. If the request is made from the board, keep the default to select all schools, or use the standard select schools function to select individual schools.

 

3. A task will be submitted to create the file. You may leave the application because an automatic notification will appear when the file is created. The notification will link you to the created file. This file may be downloaded, inspected, and, if satisfactory, may be submitted to the Ministry’s OnSIS website as directed by your board.  

Hint: select the folder icon to jump to the file manager location of Provincial > OEN > Request, which is where the file will be created. Note that this location also contains OEN request files that are submitted to the OEN website. This option will create files with OnSIS in the filename, so that they may be distinguished from the OEN files.

4. After the file has been submitted to OnSIS, and it is processed, download the results file from OnSIS.

5. From the Validate OENs in OnSIS card (shown in step 1), select the PROCESS RESULTS button to access the Process results file page.

 

6. Select the folder icon to the left of the “Location of OEN file” to access file manager. It will open file manager at the Provincial > OEN > Results folder as shown below. Use the Add icon in the app bar to add the file downloaded in step 4.  

 

7. Select the file that you want to be processed. Filenames shown below are examples only, however they must have the xml extension to appear. These result files typically have OnSIS in the file name to distinguish them from the OEN request result files.

 

8. Select the PROCESS RESULTS FILE button in the bottom bar, as shown above.

9. A task will be submitted to process the file, which will automatically provide a report indicating students with invalid OENs. You may leave the application because an automatic notification will appear when the file is created. The notification will link you to any reports generated from this process. Reports are also available in File manager in the Provincial > OEN > Results folder.

10. Analyze the results in the report (as discussed in step 9) and resolve any OEN issues.
