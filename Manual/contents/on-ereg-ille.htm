International & Indigenous language elementary program (ILLE)

 

 

 

The IILE programs option provides Ontario schools and boards with the ability to generate the official Register of Enrolment in the International & Indigenous Languages Elementary (IILE) Program. This is a classroom-based register used to record enrolment and attendance for day school pupils in Junior Kindergarten to Grade 8 who are enrolled in the IILE program.

 

This register option is available to Ontario boards and schools set up with ‘school operating type’ of continuing education. It is also available to those schools with grade 8 or lower whose school configuration is set up such that they also require access to continuing education registers.  

 

The courses must be provided through classroom instruction:

 

  • During the school year
  • In the summer

 

IILE courses may be offered:

 

  • After school
  • Before school or lunch time
  • Integrated extended-day program
  • Late-afternoon and evening program
  • Weekend
  • Summer

 

This register may include pupils enrolled in courses that are in languages other than English/French in grade JK to 8 in a publicly funded elementary school or a private school.

 

Schools and boards should be familiar with the official Ministry enrolment register instructions for continuing education programs. These instructions are updated annually on the Ministry’s site http://www.edu.gov.on.ca/eng/policyfunding/forms.html.  

 

Please be familiar with the Enrolment > Enrolment register info option in the student’s record. This will provide detailed information on the enrolment register(s) the student will appear on.

 

Hint: on the FTE and/or ADE card in the student’s record, there’s a Details section that may be expanded to show info on how FTE/ADE is calculated, and how the student’s data impacts the register. IILE classes are reported as ADE.

 

Set up classes that need to appear on the Register of Enrolment in the International & Indigenous Language Elementary (IILE) Program

There are no Ministry defined course codes for IILE programs.  A course will need to exist, using any course code that is appropriate for the board/school. It may be a generic IILE course, or multiple course codes may be created for each international or indigenous language course taught and it may optionally incorporate the grade level of the students taking the course.  A class will need to be created for each unique group of students enrolled in the IILE program.

 

If a class is to appear in the International & Indigenous Languages Elementary Program register, it must be set up (Setup > Class info > Classes) as follows:

 

  • The language of instruction must be one of the following (Ministry code must be set up correctly by the board using Setup > Class info > Languages of Instruction). Refer to the Ministry's reference tables for the extensive list of LANGUAGE_TYPE values that are categorized as International or Native. Here are some examples:

 

01 = French Creole

03 = Albanian

04 = Arabic

09 = Cayuga

10 = Cree

OTH = Other

 

  • The course offering type must be set to:

- International & Indigenous Languages Elementary (IILE)

 

  • Set the time of day type to one of the following:

- After School

- Before School/Lunch Time  

- Extended School Day

- Later-Afternoon/Evening Program

- Summer

- Weekends

 

  • Enter the most appropriate Delivery and Course delivery type

 

 

  • Since funding is based on attendance, make sure that the class schedule, start and end dates, and period times are accurate. Class start time will determine whether to use the student’s morning or afternoon attendance entry to populate this register (start time before noon will read AM attendance, and a start time after noon will read from the PM attendance).  If the school has only 2 periods, the class should be set to either period 1 (attendance will read from the AM attendance) or period 2 (attendance will read from the PM attendance).

 

Make sure each student’s add/drop dates are accurate (add date is the first day attending the class, drop date is the first date of non-attendance in the class).

 

Teachers track classroom attendance

Teachers are responsible for taking accurate classroom attendance, just like any regular class.  The class schedule, start/end dates, and the school’s calendar will define what dates the teacher is expected to take attendance.

 

Cancelled instructional sessions – Funded cancelled session “C”

Where an instructional session has been cancelled owing to exceptional circumstances (such as bad weather or an emergency) and the instructor is paid, the session will be recognized for funding purposes. These sessions are included in the total number of sessions recognized for funding. The funded cancelled session must be marked with a “C” on the register and the reason for the cancellation must be recorded. To cancel a funded session, follow these instructions:

 

  • Use Attendance > Enter attendance > Group Excursion.  Select the date and group of students (e.g. all students or a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option c Cancelled.

 

 

  • Record an attendance note describing the reason for the cancellation.
  • Apply (save) the attendance for the group.

 

Note: If the “C” session is rescheduled, the instructor should not be paid for the rescheduled session and the rescheduled session must not be included in the total number of sessions recognized for funding purposes. See Rescheduled instructional sessions.

 

Cancelled instructional sessions – Unfunded cancelled session “D”

Where an instructional session has been cancelled (I.e. the teacher is unavailable) and the instructor is not paid, the session will not be recognized for funding purposes. Do not include this session in the total number of sessions recognized for funding. The unfunded cancelled session must be marked with a “D” on the register and the reason for the cancellation must be recorded. To cancel an unfunded session, follow these instructions:

 

  • Use Attendance > Enter attendance > Group Excursion.  Select the date and group of students (e.g. a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option $ Cancelled Not Funded.

 

 

Note: If the “D” session is rescheduled, the instructor should be paid for the rescheduled session and the rescheduled session must be included in the total number of sessions recognized for funding purposes.

 

Rescheduled sessions  

If a new session is to be added to the class, typically the class end date is extended to accommodate the extra session. This is done using Setup > Class info > Classes.

 

If the rescheduled session is due to a session that is given the code “C” (funded cancelled session), the instructor should not be paid for the rescheduled session and the rescheduled session must not be included in the total number of sessions recognized for funding purposes. To reschedule a session, and have it not appear in the register, follow these instructions:

 

  • Use Attendance > Enter attendance > Group/excursion.  Select the date and group of students (e.g. All students or a specific class).
  • When selecting the attendance code to be applied to the students, scroll down to the bottom of the list of attendance codes and select the option select the option ^ Non-instructional Day

 

 

  • Record an attendance note describing the reason the class is removed from the register (e.g. rescheduled session for the mm/yyyy funded cancelled session due to weather).
  • Apply (save) the attendance for the group.

 

Generate the Register of Enrolment in the International & Indigenous Languages Elementary (IILE) Program

To access the register, select Ontario reporting > Enrolment registers > IILE programs.

 

 

The form will look like this:  

 

 

The list of classes will default any classes that are set up to appear on this register. You may limit the classes using the filters, or select an individual or range of classes to print.  A separate register is generated for each selected class.

 

Select the Print button located at the bottom of the form. Once the report has been submitted to the task queue, you may leave this application or generate other reports. You will receive notification when your report is ready. Reports are saved in file manager under Provincial > Enrolment registers.

 

 

Hint: See previously created registers by selecting the folder icon to open file manager at this location.

 

The info generated on the register may also be exported in a CSV file by checking the option shown below. The CSV file will be generated along with the register report, as part of the submission to the task queue.   

 

 

Here is an example of the register report:

 

 

The top of the report will include the register name, school year, class (course code & section letter and course title).

 

 

For each pupil in the class, it will include the following info from the student’s record (in the Personal area):

 

 

The ‘Day school pupil’ column is populated with either FT (full-time), PT (part-time) or blank (student is not a day school pupil) based on the student’s field called ‘Con. Ed. Student type’ (in student record under Enrolment > Enrolment register info).  All pupils of the board are funded and their attendance sessions are included in the non-credit ADE calculations, so this column may be ignored (it will be blank for elementary schools offering IILE since these students don’t have a Con. Ed. Student Type).

 

FT appears if the student’s Con. Ed. Student Type is one of the following types:

  • Also Attends Day School (Full-time)
  • Full-Time Day School (exclude from Register totals)

 

PT appears if the student’s Con. Ed. Student Type is one of the following types:

  • Also Attends Day School (Part-time)
  • Part-time taking same course(s) in day school (exclude from register totals).

 

Blank appears if the student’s Con. Ed. Student Type is:

  • Only Attends Con. Ed. School

 

The ‘Pupil’s board resident status’ column will show OP if the student is considered an Other Pupil (OP). OP will appear if the board defined student board residence status does not use ministry code 01, 06, 07, or 09. If a student is an “Other Pupil”, their sessions will not be recognized for funding.

 

For students also attending day school, the ‘Day School Address’ column will use the student’s origin school as recorded in the student’s record in Enrolment > Student enrolment.

 

To the right of the student info it will show the student’s attendance at sessions. This info is based on the entry of attendance by the teachers. The dates are based on the setup of the class.

 

 

For each date, a blank will appear if the student is present.  If the student was not present at the session, one of the following 4 codes will appear:

 

A      Pupil absence from a session

—     Late admission or early withdrawal (based on the student’s class add/drop dates)

C       Funded cancelled session

D       Unfunded cancelled session

 

* if the student is absent 3 or more consecutive sessions, these absences are not funded (they are excluded from total). Cancelled sessions (“C” or “D”) do not ‘break’ the consecutive absent count.  For example, if a student had 5 consecutive absences in a row (Sept 26 to Oct 15), as well as 3 consecutive absences on Jan 30, Feb 6 and Feb 11. The cancelled session on Feb 4 does not count as a consecutive absence, but it also does not break the count of consecutive absences. Therefore, if there was a possible 40 sessions that the student could attend, the school will receive funding for 32 sessions (40 - 5 - 3 = 32).

 

After the student details page, a summary page is included with an area for the principal to certify the register.

 

 

The ‘Hours per session’ is based on the daily average minutes of instruction set up for the class (using Setup > Class info > Classes). This may be based on period times, or it may be manually entered directly into the daily average minutes of instruction field. The minutes will be converted to hours on this register.

 

Funding - OnSIS reporting of classes on the Continuing Education Registers

All full and part-time taking IILE courses are eligible for funding. The FT and PT pupils who enrolled and attended sessions are included in the totals used to calculate funding. The funding allocation is based upon the total number of hours of funded classroom instruction. IILE funding is capped as follows:

 

  • During the school year at 2.5 hours of classroom instruction per week per language; and  
  • During the summer at 2.5 hours of classroom instruction per day per language.  

 

When the board class average is 23 or greater, maximum funding is generated per instructional hour. Where the average size of an international languages class is less than 23, the rate per classroom hour is reduced by $1 for every pupil less than 23.

 

Funding is based on the information submitted to OnSIS.  Regular schools requiring continuing education registers should refer to OnSIS > Continuing education for day school > IILE/Literacy & numeracy).  Continuing education schools should refer to OnSIS > Night/continuing education school > ConEd/Night school info.

 

 

From the Ministry’s instructions:

 

Elemental data is collected once a year on August 31 for the entire school year (September-August). Submit data through the Continuing Education and/or Summer School submissions (School Special Condition Type “N – Continuing Education” or “8 – Summer”) even if the IILE course is offered in a regular elementary day school (I.e. integrated extended day program).

 

As data is collected in aggregate, the pupil’s OEN is not required. Course Offering Type will not be collected for IILE programs.  

 

The following information must be reported by international or Indigenous language:

 

  • Total Number of Classes,  

  • Total Number of Instructors,

  • Total Number of Students, and  

  • Total Number of Hours

 

Number of Classes must be reported to OnSIS by the time of day the classes were offered, using the following categories:

 

  • 1. After School,  

  • 2. Before School or Lunch time  

  • 3. Integrated Extended-Day Program  

  • 4. Late-Afternoon and Evening Program  

  • 5. Weekend  

  • 6. Summer

 

ADE calculations for classes on the Continuing Education Credit Courses registers

The calculation of the average daily enrolment (ADE) is calculated as follows:

 

ADE = # of sessions recognized for funding x hours per session / 950

 

Other reports

Continuing education schools may also want to consider running these options:

 

Ontario reporting > Continuing education > Continuing education ADE

 

This generates the OnSIS continuing education enrolment detail and summary report showing average daily enrolment (ADE) info for continuing education or summer school students.

 

Ontario reporting > Enrolment registers > Course lists for grant purposes

 

This generates class by class details of each register, showing total ADE by register and for the school/board.
