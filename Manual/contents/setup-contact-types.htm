Setup > User-defined fields > Contact types

Overview

The Contact types table lists all the types of contacts that may be assigned to a student’s contact. Example contact types may be Mother, Father, Grandmother, Babysitter, Step-Father. Contact types are centrally defined by the board/division to ensure consistency. 

Clicking GO will bring the user to the following interface. 

Selecting any record in the table (e.g. row click) will present 2 options: Edit and Details.   

*NOTE: if logged into a school, Edit option is NOT available. See the section “Row click – Editing table records & Details” for further instructions. 

CLOSE button on the bottom bar will return the user to the setup page (landing page with all the cards).

 

Adding Table Records - Board/Division ONLY

If logged in from the school, no add icon is shown. 

If logged in from the board/school, the Add icon appears in card tool bar if user has edit rights. Users with view only rights do not get the Add icon. 

Click Add icon. Standard Add interface will appear with fields in the following order (field names are bold – do not include #s or colon), and the card title is called Add contact type.

1. English/French: Default screen language. This controls whether the user is entering the English or French version of a field that has unique values for each language. Fields with underlines have unique values for each language. 
 
2. Contact type title*: alphanumeric field, max character count = 250. Contact type title in English or French must be >0 characters in order to save. 
 
3. Identifier*: alphanumeric field, max character count = 7 
 
4. Effective date: date field – cannot be after expiry date 
 
5. Expiry date: date field – cannot be before effective date

SAVE and ADD buttons are disabled until the required fields exist.  

SAVE button is enabled when all required fields are entered. Selecting SAVE will save the new entry and return the user to the table. 

ADD button is enabled when all required fields are entered. Selecting ADD will save the new entry and then open a new (empty) Add interface (so user can add another record to the table). 

CANCEL button is always enabled. Selecting CANCEL will not save the new entry. 

Selecting the back arrow on the main app bar should prompt the user to save new entry IF ALL required fields are entered.

 

Row Click - Editing Table Records & Details Option

From the table, clicking any row with table records (row click) will give 2 options: Edit (make it first option) and Details.

From a school, row click will present the Details option only.  Edit option is not available for schools.  

Selecting the Details option will open a standard Details form.

CLOSE or X will close the details form and return to the table (at the same page where user was).

Selecting the Edit option will open a standard edit form. 

On the Edit form, show the DELETE icon in the app bar. 

The following fields will be visible, just like the Add form:

1. English/French: Default screen language. This controls whether the user is entering the English or French version of a field that has unique values for each language. Fields with underlines have unique values for each language. 
 
2. Contact type title*: alphanumeric field, max character count = 250. Contact type title in English or French must be >0 characters in order to save. 
 
3. Identifier*: alphanumeric field, max character count = 7 
 
4. Effective date: date field – cannot be after expiry date 
 
5. Expiry date: date field – cannot be before effective date

PREVIOUS (if there is a previous row in the table) and NEXT (if there is a next row in the table) buttons will appear.   

Selecting PREVIOUS button will save the edits and then reopen the edit form with the previous record (row) in the table. Selecting NEXT button will save the edits and then reopen the edit form with the next record (row) in the table.

Deleting Table Records

Editing a table record (row click > edit option), will show the delete icon. This icon will be disabled if the role has been assigned to any staff. 

DISCARD button will remove the record from the table. CANCEL will retain the record in the table.
