Scheduling next year - Secondary - Part I: Beginning steps

 

The following steps are fundamental steps to scheduling a secondary school. These steps are generally done once, at the start of the scheduling process. Follow the links for detailed instructions.

 

1. Initialize next year

2. Check lunches

3. Check sem/term/day/period info

4. Check courses

5. Record course selections

6. Analyze course selections

7. Add/remove classes

8. Add teacher, room, term & day/period constraints

9. Add base master timetable with constraints

10. Create snapshot of base timetable

 

 

 

 

1. Initialize next year

The scheduling process beings with running the Scheduling next year > Prepare for next year > Initialize next year process.  This process duplicates the school information from this year, into the next year.  This makes re-entering last year's homerooms, advisor groups, courses, classes, and semester/term/day/period information unnecessary if they will be similar for the next year.  It also advances the students’ next year grades.

 

2. Check lunches

Check that lunch periods are correctly defined, if needed, using Scheduling next year > Prepare for next year > Lunches.

 

3. Check sem/term/day/period info

Check that the number of semesters, terms, periods, and days in cycle are set up correctly for next year using Scheduling next year > Prepare for next year > Sem/term/day/period. It is very important to make sure the sets and patterns are set up correctly as explained in Important Concepts.

 

4. Check courses

Check that the list of courses is up to date for the courses offered next year. Brand new courses may first need to be added to the codes table by the board/division/district.

 

5. Record course selection

Students typically enter their course selections through edsembli CONNECT or a career planning application (if integration is setup). Those course selections should appear in the edsembli SIS without any action required on the part of the school.  If students are unable to enter course selections, they must be added by the school using any of these options:

 

  • Student record > Class info > Course selections

 

  • Scheduling next year > Course selections > Individual student or Add/delete/replace course selections.

 

 

6. Analyze course selections

Analyze course selections by running these options:

 

  • Use Scheduling next year > Course selections > Course tally to sort by the number of requests column to identify courses with low interest. If a course is cancelled, use the Scheduling next year > Course selections > Counselling list option to identify the affected students, and make changes to their course selections.

 

  • Further analyze courses using Course analysis by grade, Invalid prerequisites and List course selections. Some schools have guidance staff 'approve' each student's courses.  This feature is available within the Course selections option in each student's record or using the List Course selections option.

 

 

7. Add/remove classes

Use the class analysis option to create exactly the number of classes needed to meet the needs of next year's students based on their course selections. It can be used to quickly add and remove classes to the school's list of classes.

At this point, if there are any changes to rooms for the next year, they should be made to the list of room using Scheduling next year > Classes > Rooms.

8. Add teacher, room, sem/term, day/period constraints

Constraints are the class attributes that the school need to ensure are taken into account as the school's timetable is created.  The fewer constraints recorded, the more flexibility the scheduling tools have in finding the best placement for a class (the location with the fewest conflicts). These constraints must be recorded by editing the classes with the constraints using Scheduling next year > Classes > Classes:

 

  • If a class must be taught by a specific teacher, make sure the teacher is recorded for the class.

  • If a class must be taught in a specific room, make sure the room is recorded for the class.

  • If a class must be taught in a specific semester/term pattern, lock the class into that pattern (example below)

  • If a class must be taught in a specific day/period pattern, lock the class into that pattern.

 

You may lock the semester/term pattern and/or the day/period pattern for any class by selecting the lock/unlock icon (while editing the class).  If you want to ensure that a class remains in a specific semester, then lock the semester/term pattern.  Similarly, if you want to ensure that a class remains in a specific period, then lock the day/period pattern.  The automatic scheduling tool will not change those setting if they are locked.

 

 

 

9. Analyze base master timetable

Before using the tools to schedule classes, it is good to have a 'base' timetable that represents the master timetable prior to scheduling classes.  Sometimes, conflicts are introduced into the timetable before you even begin using the scheduling tools.  Rather than try to figure out problems after scheduling more classes, it is a good idea to see whether your class constraints are causing any problems.  To analyze the base master timetable, follow these steps:

 

  • Check that class schedules are locked only where absolutely necessary. Hint: you can sort the list of classes by the lock columns.
  • Check that classes have the correct semester/term and day/period set (e.g. make sure a class doesn't have a full year set when it is supposed to have a single semester set; or make sure a class doesn't have a double period set when it is supposed to have a single period set).
  • Use Scheduling next year > Classes > Reset class schedules to clear out the patterns for classes that are not locked (and make a snapshot when prompted as a precaution).  The scheduling tools will assign the patterns that are best for the class.  
  • Run Scheduling next year > Assign students to classes > Load students using the default options.
  • Use Scheduling next year > Analyze master timetable > Analyze conflicts and look at the results in the analyze conflicts card. This will show courses that have classes scheduled into the timetable (e.g. class has both a semester/term and day/period pattern). Look a the Unassigned student column. This will indicate the number of student who were not able to get into this class. Depending upon the constraints in the timetable, at this point there should be very few conflicts (low number of unassigned students). Select the row to access the Schedule option for each class (e.g. it will open master builder - manual for that class and you can see if there is a better location if you remove one of the constraints).  If any changes are made to any classes, repeat the load student and analyze conflicts steps until you are satisfied with the base timetable.

      

10. Create snapshot of base timetable

Taking a snapshot of this base timetable will ensure that there is always the ability to start all over again, without having to repeat the earlier steps. Create a snapshot using Scheduling next year > Classes > Snapshots.

 


At this point, you are ready for Part II: Scheduling.
