Overview

The setup of a school’s semesters, terms, days and periods is generally is done during the initial set up of a new school and rarely needs to be revisited. The exception would be if a school makes changes to the core schedule (i.e. changes in any of the parameters that define when classes are taught):

 

  • Number of semesters in the school year
  • Number of terms per semester
  • Number of days in the cycle
  • Number of periods per day
  • Start and end times of each period in each day
  • Changes to the tumble/rotation of classes

 

Semesters/terms/days/ and/or periods setup is available for this year and next year. The existing set up is automatically copied to next year. Any changes that you make for the next school year will not impact the current year.

 

There is a separate class for each time a course is taught to a different group of students. “When” each class is taught is called  the class schedule. The class schedule provides information that lets the class fit into the student and teacher timetables. In order for all the classes to fit into the timetables, classes have both Semester/Term Sets and Day/Period Sets with associated patterns.

 

There are sets with patterns for when a class is taught with respect to the semesters/terms (duration), and there are sets with  patterns for when a class is taught with respect to the day and period (rotation).

 

A set is a main grouping that contains all the appropriate possible patterns and is  a critical attribute if you plan to use any of the automatic scheduling features.  A set contains patterns, and when building a timetable, you indicate the appropriate set for a class and let the timetable building application determine the best pattern that belongs to that set or you can manually assign the desired pattern. Please see below Understanding sets and patterns for a more detailed explanation.

 

From the main menu select Setup > School. This will take you to the School section of the setup page where you will find the Semesters/terms/days/periods card. The board will not see this card.

 

 

Select either THIS YEAR or NEXT YEAR.

 

Note: this card is also available from the Scheduling next year > Prepare for next year. When accessed this way, the card will provide a GO button and it will only modify next year's sets and patterns.

Before using the setup application you should first understand the terminology and intended usage of the sets and patterns for semester/term and day/period. Here are some examples.

 

Understanding semester/term sets and patterns

This areas refers to the ‘duration of a class’. How long it lasts/spans during a school year. If a school has some full year classes (September to June), many classes that are taught for a single semester (half the year) and some classes that are taught for a quarter of the year (or half a semester), the school would typically be set up as 2 semesters, and 2 terms per semester.

 

In this example there would be three Sets (Full Year, Semester and Term) each with their own unique and appropriate patterns. For example, the “Semester” set would have 2 patterns because there are 2 possible schedules: semester 1 or semester 2. Sample set and pattern names might be:

 

Set = Semester (main grouping)

 

Patterns: Sem 1 and Sem 2 (possible times within the group when classes can be taught. Either Semester 1 or Semester 2)

 

 

The “Term” set would have 4 patterns because there are 4 possible schedules: semester 1/term 1, semester 1/term 2, semester 2/term 1, semester 2/term 2. Sample set and pattern names are often short but meaningful.

 

Set = Term

Patterns = Sem1Term1, Sem1Term2, Sem2Term1, Sem2Term2.

 

 

The “Full year” set would have only 1 pattern because it has only 1 possible schedule. The set and pattern name can be the same or different, but both a set and a pattern must exist. If only 1 pattern exists, it will be automatically given to a class with this set.

 

Set = Full year

Pattern = Full year

 

 

Understanding day/period sets and patterns

This areas refers to the ‘rotation of a class’. How it tumbles or rotates within a cycle. In this example, the school has 4 periods each day and runs on a 2 day cycle. Most classes are assigned to a single period that is taught on both day 1 and day 2. However, the school has some classes that are taught every other day (alternating day classes) and some classes that are taught in for the morning or afternoon for a double period (i.e. morning co-op).

 

For example, let’s set up for the classes that make up the majority of the classes, the regular rotation and we’ll call the set the Regular. This set would have 4 patterns (possibilities) because there are 4 possible schedules into which a class could fall: Period 1, or Period 2 or Period 3 or  to Period 4. Sample set and pattern names:

 

Set = Regular

Patterns:  Period 1, Period 2, Period 3, Period 4

 

For each pattern, all the periods for each of the days are presented (days in columns, periods in rows). For each day in the cycle, the periods, where this pattern is taught, are checked off as shown here:

 

  

Once all the patterns are defined, they will appear like this (note: only Period 1 and Period 2 patterns are shown here):

 

 

The half day set would have 2 patterns because there are 2 possible schedules: mornings or afternoons. Sample set and pattern names are often short but meaningful.

 

Set = Half day

Patterns = AM, PM

 

The alternate day set would have 8 patterns because the class may be taught in one of the 4 periods on day 1 or one of the four periods on day 2 (e.g. period 1 day 1 or period 1 day 2).

 

Set = Alt day

Patterns = P1D1, P2D1, P3D1, P4D1, P1D2, P2D2, P3D2, P4D2

Defining period times

Be sure to accurately define period times, i.e. the period start and end. The period times may be different on each day in the school’s cycle, but often they are the same.

 

The period times are used in any reports that require a count of the minutes of instruction for classes. These times are also used to determine a student’s current location by comparing where the student should be vs where the student actually is.

 

In the setup application, select the DAY/PERIOD tab to see the Period times card (located below the Defined period times section which is collapsed in the example below).

 

 

Use the Edit icon on the Period times card (shown above), to set the start and end times for each period. Times must be in chronological order, but the end time of one period may be the same as the start time for the next period, or the next period may be later to account for travel time between classes.

 

 

Once you have defined the period times for day 1, those times may be copied to the other days by selecting the More icon on the edit period times card to access the All days alike option. You can also print by selecting Print from the More icon.

 

If the period times are different between each day in the school’s cycle, use the tab strip to move to a different day to define unique start and end times for that day.

Unique class schedules

Classes without periods (floater classes): Many schools create a day/period set and pattern called Floater (or Unscheduled or Period X or any other meaningful name) for classes that are assigned into a semester/term, but are not scheduled into any period. These classes can have achievement data but they cannot have attendance stored because they have no period (something to consider for students with the 'by period' attendance method).

 

 

Multiple schedules (multiple occurrences): Each class’ schedule is defined in the class. There may be times where a class has a unique situation where they require more than 1 schedule to accurately indicate when a class is taught. For example, if a class is taught all year long, but it changes from period 1 to period 2, it is possible to set up 2 schedules in the class without the need to create a unique set/pattern. Multiple schedules for 1 class are sometimes referred to as occurrences.

 

Schedule 1:  Semester/Sem 1, Regular/Period 1

Schedule 2:  Semester/Sem 2, Regular/Period 2

 

Special schedule: There are also times where the definition of sets and patterns is simply not practical for a school. This is often the case where a timetable is manually created without any formal patterns. For these situations, the class’ schedule may be defined directly in the setup of the class rather than creating a unique pattern here.  In the set up of the class, it may be flagged as requiring a special schedule which then allows the user to select the specific semesters/terms and days/periods unique to that class. These classes will be automatically listed in a card called Special Schedule located at the bottom of the semester/term and day/period setup page. Patterns cannot be added/edited here.  Note: Any automatic timetabling tools cannot schedule these types of classes.

 

 

Notes for new schools

When adding a new class is created, the class schedule will default the first set alphabetically, and when selecting sets and patterns from drop down lists, the names will be listed in alphabetical order.  For this reason, some schools add numbers in front of the set and pattern names to force a specific order, while most schools leave the names as they want.

 

Typically, new schools will have sets and patterns automatically created as data is imported from other sources, however the names of the sets and patterns may be changed to better suit the terminology used in the schools. See Editing semester/term sets and patterns or Editing day/period sets and patterns for additional information.
