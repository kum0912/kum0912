Scheduling next year > Assign students to classes > Load students

 

This tool generates next year’s timetables for students by assigning students into specific class sections of the courses that the student requested.  Students are randomly loaded into classes based on the semester/term and period/day information assigned to each class.   

 

Every time you run load students, the timetables will be analyzed to ensure that there are no significant problems before scheduling another group of classes. The following processes are repeated multiple times to build the best timetable for the students and the school.

 

Choose Scheduling > Assign students to classes then select Go on the Load students card.

 

 

The main Load students page will appear with two options to consider:

 

 

Once you set options, select the LOAD STUDENTS button in the bottom bar.

 

Limit class size to set maximum: Check this option off if you do not want to overload classes. Each class has a setting for the number of seats (the capacity of the room). If you check this option, then students will not be assigned into classes if they are full.   

 

If you check this option, then students will have conflicts (unable to enrol into a requested course) if the class fits into the student’s timetable but the class is full. The Course conflict analysis report will identify these students as conflicts, and if a large number of students are unable to enrol, you may consider running Master builder – manual on the class section with the fewest number of students to see if there is a better location for the class.  

 

Hint: if you uncheck this option, carefully examine the actual class size after each load. For example, if most classes have a class size of 30 seats, and you allow classes to be overloaded, after each load, look at the classes table and sort classes by occupancy. Classes that are overloaded are actually assigned into a very good period. Look at the other class sections of the course and consider running Master builder – manual on the classes that have low occupancy.  

 

Substitute conflicting preferred course selections with alternate courses: Each student’s preferred course selection has an optional check box for “Can this course be substituted with an alternate?”. If this is set to yes for a student’s preferred course, and the load students tool is run with the substitute question set to yet, then the load students tool may generate a different timetable for the student. The student’s timetable will be generated as usual, however, if a preferred course cannot be scheduled due to a conflict (or the class is overloaded), it will attempt to create a timetable using the student’s alternate course selection instead.

 

After the LOAD STUDENTS process is complete, the students will have some/all of their classes in their next year’s timetable. See Analyze student timetables/conflicts for the next steps.
