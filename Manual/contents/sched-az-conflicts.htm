Analyze conflicts

After a group of classes are scheduled, and the students are loaded into timetables (using load students), some students may have timetable conflicts. A conflict exists if the student is unable to be assigned to a requested scheduled course.

While the timetable is being created, it is important to analyze the conflicts after each group of classes is scheduled and the load students process has run. The Analyze conflicts tool is step 3 in the scheduling process that is repeated for small groups of classes:

Rather than looking at student conflicts, one at a time, the most efficient way to analyze the resulting timetables is to use the Analyze conflicts option. This option helps identify the courses that have large numbers of students unable to get assigned into classes.

To analyze conflicts choose Scheduling next year > Analyze master timetable.

 

Select GO on the Analyze conflicts card and the following screen will appear:

For every course with at least 1 scheduled class, the following information is provided:

Scheduled classes: the number of class sections that have been scheduled.

Preferred requests: the number of students requesting the course as a preferred course selection.

Unassigned students: the number of students who were not able to get assigned into a class of the course.

Percent unassigned: the percent of students requesting the course who are unassigned.  This is the number of unassigned students divided by the number of preferred request multiplied by 100. For example, if a course has 50 requests with 10 students not assigned a class, it would show 20% unassigned.

Unscheduled classes: the number of class sections that have not yet been scheduled. If there are classes not yet scheduled, then unassigned students will decrease when these classes are scheduled. Typically, if there are unscheduled classes, you can ignore these courses.

 

By default, the courses with the greatest number of unassigned students are listed first, but the list may be by any column order by selecting the column title. If the course has 0 unscheduled classes, and a high number of unassigned students, it may indicate a ‘problem’ course. Here’s a suggested approach for these ‘problem’ courses to see if there is a better placement in the schedule which would allow more students to get assigned into a class:

  1. Select the icon to change from hiding to showing class info:

  2. The grid will change to list each class, along with occupancy (number of students in the class).

  3. Select (row click) the class with the lowest occupancy.  This will provide a quick link to schedule the class (using the Master builder – Manual tool). This is where you can identify if there is a different placement that would have fewer student conflicts. The automatic master builder is limited to working within the constraints placed on the class such as locking a class to a specific semester. It will also indicate if there is a better location for students if a different teacher or room can be assigned.

 

Below the Analyze conflicts card, you may see an Unscheduled classes card. Detailed information is not provided for courses that have no classes scheduled yet. They are omitted from the Analyze conflicts card, however, they are listed in the Unscheduled classes card, which is collapsed by default.  When you are done the scheduling process, there should be no unscheduled classes.  Expand the card to confirm that there are no unscheduled classes.
