Overview

The Personal menu option in a student’s record contains student-specific information, i.e. information that is the same regardless of what school the student attends. If your school is the record owner, then you can edit this information. Otherwise you cannot edit.

 

Choose Personal > Name from the submenu.

Name information is grouped and presented on a separate cards titled Name, Aliases (AKA) and Information disclosure restrictions (Alberta students only), as described below.

Name

The Name card contains the student’s preferred (usual) names as well as the exact legal names as provided on an official source document. For most students, the surname field is the same as the student’s legal surname.  

 

 

As the student’s name is edited, previously recorded names are maintained in the Name changes area for reference only (and therefore may not be edited). Name change information is collapsed by default on the card, but may be expanded if you want to see the student’s previous names. When searching for students, the search may optionally consider matches based on the information in the name changes area.

 

There is a Y/N  switch to indicate whether the legal surname is the same as the surname. When the switch is set to Yes, legal surname and suffix fields are not available.

 

 

If you set the Legal surname same as surname switch to No, then you can fill in legal surname and suffix fields.

The following fields are recorded as part of the student’s name:

 

Surname:  The student’s preferred surname.

 

Legal surname same as surname (Y/N flag):  If no, legal surname field and Suffix fields are available.

 

Legal surname: If flag above is N, this field is enabled for the entry of the student’s legal surname as provided on an official source document. If the flag above is Y, this field does not appear.

 

Suffix: If flag above is N, this field is enabled for the entry of the student’s suffix (that appears after the surname such as Jr., Sr., III). If the flag above is Y, this field does not appear.

 

Given name:  The student’s first name as provided on an official source document.

 

Middle name(s):  The student’s middle name or names as provided on an official source document.

 

Usual name:  The student’s preferred first name or names. It will default the student’s given name.

Setting this to No, indicates that you are not able to enter the student's name exactly as it appears on the Source Document. Aliases (AKA)

A list of alias names that a student may use can be maintained. Aliases are names that a person may also be called, and sometimes they are referred to as AKA (Also Known As). Each Alias name may have a surname, given name(s) and middle name(s). As long as something is entered in the surname or given name fields, the alias name will be valid. Multiple alias names may be recorded for a student. When searching for students, the search may optionally consider matches based on aliases.

 

 

To edit and add an alias use the edit icon and this Student records form will appear:

 

 

To edit an alias select the edit icon. To delete and alias access the edit form and select the delete icon. You'll be asked to confirm the deletion.

 

 

 

To add an alias use the add FAB. The form has the same fields as found on the edit form.

 

Information disclosure restrictions

Alberta Students will have an Information disclosure restrictions card, although for most students it will be empty. Information disclosure restrictions is defined below. If a restriction applies to a student, it must be added. A student may have multiple restrictions, with the following fields recorded for each:

 

Restriction type: Select the type of information disclosure restriction from the list.  See below for a description of the 3 categories of restrictions.

 

Effective Date: The date the restriction started. A date must be entered.

 

Expiry Date: The date the restriction ends (if applicable).

 

Restriction Details: An optional note to provide additional details.  

 

Alberta Educations description: Under normal circumstances, parents/guardians have a “right of access” to their student’s record. This means that in addition to the student, as well as those working within the education system that have access on a need-to-know basis, the student’s parents/guardians are also allowed access to the information contained within the student record.

 

However, there are situations where these normal rules do not apply such as when a Court Order prohibits/restricts specified individual(s,) including parents, from access. Three categories of disclosure restriction have been defined to further clarify specifics for a given situation:

 

  1. Court Ordered – A restriction imposed as a result of a legal process/procedure. These are generally supported with legal documentation.
  2. Caution – A restriction put in place, but may not be supported by legal documentation. This may be because legal documentation has not yet been provided, or to restrict inappropriate access to the student’s information (for example, a note that the student’s uncle or friend sometimes pretends to be the student’s father).
  3. Independent Student Under 18 – A restriction put in place because the student has been legally declared as independent. This should be used in conjunction with Grant Program Code 121 (Independent Student) and highlights that while the student is under 18, the student’s parents are not allowed access to the student’s information without the student’s permission.
