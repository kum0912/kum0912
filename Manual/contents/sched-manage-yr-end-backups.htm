Scheduling next year > Year-end roll over > Manage year-end backups

Board/division/district only. Single private schools may skip this step.

 

Before using the Transfer to new year option, the board/division/district will need to establish the year end backup specifications for the current school year, including the naming conventions, locations, and groupings of schools that will be transferring to current at the same time.

 

There can be multiple year end backup data sets per board/division, corresponding to the times when groups of schools will be transferred. If all schools are transferring at the same time, only one backup data set is required.

 

Each backup data set is essentially a copy of the edsembli SIS database at a specific point in time. Although the data set contains all the schools, its main purpose is to provide access to the year end data (just prior to transferring to the new year) for those schools who are being transferred at the same time.

 

If there is a need to run the transfer at different times for different groups of schools, multiple data sets are required. For example, a division may want to transfer all the elementary schools at the end of June, then transfer all the secondary schools at the end of July, followed by summer/continuing education schools at some point in the fall.
