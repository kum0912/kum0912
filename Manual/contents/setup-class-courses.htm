Overview

Codes, courses and classes are closely related when a course is a provincially defined course and/or is to be recorded in the student’s “completed courses” area of their student record. The board/division is responsible for the shared list of codes, and each school is responsible for maintaining their school’s courses and classes. A course that a student takes to earn credits toward their diploma must first be centrally defined as a code. See Code fields for information on codes.

 

This section of the manual will refer to the setup of the courses table at the school. The courses table is a list of courses (subjects) that the school is teaching during the current school year, or plans to teach next year. A course must exist in the courses table before the school can create individual classes (sections/happenings) for each unique delivery of the course to the students. See Course fields for information on classes.

 

There are 2 tables of courses: one for this year and one for next year. The courses in the next year table represent the courses that students can select when making their next year’s course selections, however sometimes courses are also needed for timetable building purposes that are not available to the students.

 

The setup area for courses is used in the following situations:

 

  • The board/division has added a new course to the codes table and your school wants to start using the new course.  
  • The school wants to add a new non-credit course for its own use.  The new course will be worth 0 credits.  
  • The school is no longer teaching a course and the course needs to be removed.
  • The school wants to modify the course title defined by the board/division (in the code table). The school defined title is for school use only (course selections, timetables, class lists, etc). The official title will be saved in the completed courses area (transcript).
  • School want to add or edit course fees.
  • School wants to edit course information that is specific to this school. This information is detailed below, but includes information for timetable building such as weighting (for balancing course work loads between semesters), prerequisites, and sequencing with respect to other courses taken the same school year (e.g. course 1 and course 2 must be scheduled in opposite semesters, or course 1 must be taught before course 2).
  • Course descriptions (short and long) need to be entered/edited to assist students with online course selections.  
  • The board/division needs to edit courses as needed. Please read Understanding courses  before proceeding with any edits.

Adding a new course

Use the Add icon in the Courses tool bar to add a new course. Courses are maintained by the board/district (Central office). The Add icon is not available for schools. Schools have view only access.

Editing a course

Select the course from the list to access the Edit option as shown below.

 

 

 

Then proceed to change the course information as needed. While editing a course, it is possible to navigate to the next or previous course in the list using the NEXT COURSE and PREVIOUS COURSE buttons at the bottom of page. Selecting any of the buttons at the bottom of the page will save any edits to the course.

 

Board/division edits to the 3 key fields (course, number, or grade) will automatically change those values EVERYWHERE the course is used. This includes all students’ completed courses (transcripts), all schools’ courses and classes tables for this and next year. This is rarely required but provides an opportunity to correct mistakes in these fields.

 

Board/division edits to the other course fields (e.g. fields that are not the 3 key fields) will not impact any existing use of the course. It will impact future use only. For example, changing a credit value will only impact the credit value when a new class is created (credit values of existing classes will not change). Changing the category field (used to determine if the course fulfils graduation requirements) will not change the category for those students that have already recorded the course in their completed courses (transcripts), however, the next students who have this course added to their completed courses will use the new category.

Removing a course

A course may not be removed if any classes exist in the school’s classes table for the appropriate school year (this or next). Next year’s courses cannot be removed if the course has been entered in any student’s course selections. If a course is to be removed, select the course from the list to access the Edit option. From the edit course tool bar, select the Discard icon.

 

Course fields

When a course is added or edited, the DETAILS tab is defaulted and this is typically the tab where any edits are made. See Sequenced courses,  Prerequisites, and Report card for information on the other tabs.

DETAILS tab

The following fields are located on the DETAILS tab when editing a course:

General

Course defined in codes table: A field that must be checked if it is a credit course or any other course that is to be saved as part of the student’s completed courses (transcript).  This is rarely changed once the course is added to the school’s table. When adding a new course, it is important to understand the impact of this field.

 

If the box is checked, then the course is selected from a drop down list of codes maintained by the board/division.

 

 

If the checkbox is not checked, then the course is created for the school’s use by entering a course code, course number (PASI code in Alberta), and the grade level of the course. These 3 fields must be unique to this school’s courses table and the codes table. Credit value will be set to 0 and disabled.

 

Course information

Selection number (optional): This is an optional feature, but may be useful if a school provides students with a course listing that has a unique number assigned to each course (e.g. ADA1A is selection number 1, ADA2A is number 2, etc). This unique number is entered here. This allows the entry of a student’s course selections to be done by entering this number rather than an alphanumeric course code.

 

Display for student use during course selection: A checkbox indicates whether or not to allow students to select this course when they are entering their own selections.

Description

Is title same as code: (Only applicable if ‘Course defined in codes table’ is checked)  Indicate yes if the title defined by the board/division in the codes table is to be used.  Indicate no if the schools want to make a modification to the title. Modifications are for school use only. The official title will be stored when a course is stored in the student’s completed courses.

 

If this field is checked, then the Title English and Title French fields are enabled for edits.

 

Title English: The title of the course in English.

 

Title French: The title of the course in French, if required.

 

Short description: A short description of the course to help students when selecting next year’s courses.

 

Long description: A longer description of the course  to help students when selecting next year’s courses.

 

Department: The department associated with this course.  The table of departments is user-definable by the board/division/school.  This field is optional and is for informational purposes only.

 

Course fees

Each course in the school can be set up to charge course fees automatically when adding to timetables by defining these 3 fields for the course:

 

Fee: If a student taking this course is expected to pay a course fee, enter the dollar value of the fee in the format ##.## (e.g. 12.50 for a $12.50 course fee).

 

No refund after (days): If a student has a course fee but then drops the course, the course fee will be refunded unless there is a value entered here. If 10 were entered, the student can drop the course and receive a refund for the first 9 normal school days from the date the student started the class, however, on the 10 day or later, no refund would be applied. Fees and refunds are automatically assigned to a student as classes are added and dropped from their timetable.

 

Fee type: The fee type that will appear in the student’s account when course fees are added and refunded. Fee types are a board/division/district or school defined.

 

 

SEQUENCED COURSES tab

This feature will be available in a later release.

 

When a student’s timetable can be automatically created by the application, sometimes it is important to control the sequencing order of one course with respect to another course.  Sequencing is only applicable to secondary schools, where the school has more than 1 semester and/or term. Typically sequencing refers to semestered courses, but it also works for courses taught in single terms.

 

A course may have multiple sequenced courses added. In the definitions below, the course you are working on will be considered course A. The course added to the SEQUENCED tab will be considered course B. For each entry on the SEQUENCED COURSES, you select the course (course B) and the type of sequencing.

 

Sequence types: These are the possible sequence types that will be considered if both course A and B need to be scheduled into a student’s timetable:

 

Alternate: If course A has an alternate course B, courses A and B may not be taken in the same semester. Note: this sequencing type will automatically appear on the SEQUENCED COURSES tab for course B.

 

Blocked: If course A has a blocked course B, when the student is scheduled into the class “a” section of course A, then the student will be scheduled into the “a” class section of course B.  The students in the section “a” class of course A and B are the same students, while the students in section “b” of course A and B are the same students, and the “c” section classes have the same students, etc. This feature is used when the schools wants to keep students together for some classes during the day. Note: this sequencing type will automatically appear on the SEQUENCED COURSES tab for course B.

 

Co-sequenced: If course A has a co-sequenced course B, course B must be taken before or during the same semester as course A. For example, chemistry might have a co-sequenced math course so that the students have the math skills required for the chemistry course.

 

Pre-sequenced: If course A has a pre-sequenced course B, course B must be taken in semester 1 and course A must be taken in semester 2.

 

Teacher: If course A has a teacher sequenced course B, when the student is scheduled into the course A, then the student is scheduled into course B taught by the teacher of course A. Note: this sequencing type will automatically appear on the SEQUENCED COURSES tab for course B.

PREREQUISITES tab

Prerequisites define the courses that a student must have already successfully completed prior to taking the course. Prerequisite information is used when a student’s course selections for the next school year are entered. Students may not select courses that they have not yet fulfilled the requirements considering the student’s completed courses and courses that are expected to be completed by the end of the current year, as well as the other course selections the student has already requested. School staff may enter a course selection for a student who does not yet have the prerequisites, as they will be given a warning, but they may opt to continue with assigning the course to the student. Prerequisite information is also checked when classes are entered into a student’s timetable, and again, school staff may proceed past any warnings.

Note to Alberta schools: When PASI is enabled, the prerequisite information is populated automatically from the provincial PASI core database as part of the synchronization  process. Prerequisite information from the province is not editable, however the division may opt to add additional prerequisite information that applies to the students in their division. Prerequisites may also be added in each school’s course table, if needed. The school’s course table will automatically include the prerequisite information from the province, as well as prerequisite information from the division. Schools may not remove or edit information that is automatically populated.
