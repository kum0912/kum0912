OnSIS > Submission period > Setup

 

 

Each submission period includes a setup card. This card will provide quick access to the file manager location where the OnSIS extracts (if applicable), batch files, and results files are stored for the submission.

 

The setup may also contain options that may influence the OnSIS batch files, so it is good practice to check this option before starting any new submission period. If applicable, these setup options will appear below the file manager locations:

 

Homeroom Aliases

Grade 10 to 12 Designation Types

Campus Numbers

 

Choose OnSIS > (submission period) > Setup and select the GO button to access the setup options for the submission period.

 

 

 

The Setup form will be customized according to the submission period selected.  Some submission periods will only contain the file manager links, and others may contain setup options as explained below.  Here is an example of the form showing the file manager links and the ‘Grade 10 to 12 Designation Types’ setup option:

 

 

The file manager locations are listed at the top of the setup form. Select any of the folder icons to quickly access those files in file manager.

 

 

Extract files: these are the CSV files that come from OnSIS. They provide info about the state of the OnSIS data. If a submission period does not require extracts, this location will not appear.

 

Batch files:  these are the XML files that the SIS creates. They provide add/update instructions so that OnSIS can update their data to match the data in the SIS. These files are created by the SIS and are uploaded to OnSIS for processing.

 

Results files: these are OUT files that OnSIS creates when the batch files are processed. They provide info about whether the file is processed successfully or with errors/warnings. These files are downloaded from OnSIS and then processed by the SIS to create an easy to read PDF file of the results.

 

 

If applicable, there will be a subheader for each setup option that applies to the submission period.  Here are the possible setup options:

 

Homeroom Aliases

This setup option appears for the October, March and June Elementary School submission periods.   

 

OnSIS collects each elementary student’s homeroom in a field called CLASS_CODE. This setup option provides a way for a school to report a different class code to OnSIS than the class code used by the school. For example, if a school splits the students in homeroom 4/5 into 2 distinct homerooms (such as 4/5C for the students taking core French and the 4/5I for the students taking immersion French), this setup will allow all 20 students in the homeroom to be reported to OnSIS as one class called 4/5.  

 

 

The list of homerooms in the school will appear in the Homeroom column. The second column will also show homerooms, but this ‘OnSIS homeroom’ column may be edited so that the student’s homeroom reported to OnSIS will use something else.   

 

This homeroom aliases setup will be saved and used for all applicable submission periods. This will ensure that the same class groupings are used throughout the school year.

 

Grade 10 to 12 Designation Types

This setup option appears for the October, March and June Secondary School submission periods.

  

 

 

 

Since some schools may assign grades to students using different methods, OnSIS collects information about how a grade 10 to 12 student’s grade is assigned, in a field called GRADE_DESIGNATION_TYPE in the batch files.  There are 3 possible methods that the Ministry considers appropriate for determining a student’s grade:

 

Years of attendance:  This designation type describes schools where the student’s grade is based on the number of years the student has attended high school (e.g. grade is automatically advanced regardless of the student's academic success).

 

Accumulated credits: This designation type describes schools where the student’s grade is based on the credits earned.

 

Principal designation: This designation type describes schools where the principal designates what grade is most appropriate for the students.

 

Most schools designate the grade based on years of attendance. This may be changed by selecting the drop down arrow to select from the 3 possible types. Grade 12 students can have different designations based on credit counts. Credit counts are based on the accumulation of credits at the end of the school year.  Timetable and transcript credits are totaled for the grade 12 M, and U courses.  

 

This grade designation setup will be saved and used for all applicable submission periods. This will ensure that the same designation type is used throughout the school year.

 

Campus numbers

This setup option appears for the following submission periods:

 

  • Continuing education for day school (IILE/Literacy & numercy)
  • Night/continuing education school
  • Summer school

 

CAMPUS_NUMBER is an optional field in the OnSIS batch files for the students in the above submission periods.  This feature was provided by OnSIS to help continuing education schools with multiple campuses who report to OnSIS using one school. It allows the school to identify which campus/site the student attended. The field is for the school use only. The OnSIS campus number field will accept numbers or letters, but not spaces.

 

To populate the campus number in the batch file, the students must be identified as attending a specific campus by following these steps:

 

  1. Determine which of the 4 possible defined characters will be used for all students to identify the campus. In this example, we’ll use defined character 3.

 

  1. Students will need to have an entry in defined character 3. The entry must be the same for all students at the campus. In this example, we’ll use the value M in defined character 3 for students attending St. Mary’s campus, and the value W for the students attending Willowcreek campus. Defined characters are assigned from the student’s record using Enrolment > Flags.  Once assigned, the defined character will appear in the profile area of the student’s record as shown here:

 

  1. From the Setup option in the OnSIS submission, define the campus numbers to be included in the OnSIS batch files.  Use the Add icon in the Campus Numbers card to define the number to be sent.

 

 

Specify the number to be reported to OnSIS (it can be letters or numbers, but spaces are not allowed). Then indicate which of the 4 defined characters is used for identifying the campus, along with the value entered in the student’s defined character if the student is attending that campus.

 

 

 

All the campuses should be added to the setup.  

 

 

The campus number setup will be saved and used for all applicable submission periods.
