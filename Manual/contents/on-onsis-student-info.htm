OnSIS > Submission period > Student info

 

The Student info option will appear for the following submission periods:

  

  • October elementary school
  • October secondary school
  • March elementary school
  • March secondary school
  • June elementary school
  • June secondary school

 

This option is used to create batch files, using the CREATE button. The batch file is uploaded to OnSIS through the OnSIS web site.  Once OnSIS processes the batch file, a results file is created. The results file is downloaded from OnSIS and is then processed using the PROCESS RESULTS button.  

 

This section describes the Student info batch files. See the OnSIS Important concepts section for info on Updating data in OnSIS using the ‘Create/Upload/Process’ cycle.

 

The file provides information about the students in the school. There are slight variations for the student info option for elementary vs secondary submissions. Follow the links below for the file specifications.

The OnSIS batch file options are detailed below. Prior to selecting the CREATE BATCH FILE option, determine whether other options on the form should be selected.   

 

 

Purge:  Y/N field. See Important concepts – Purge check boxes for more info.

 

Save results to CSV file (aka Also create CSV file): Y/N field. If set to Yes, the CREATE BATCH FILE option will create the requested batch file(s) and the contents of the batch file(s) will also appear in a comma delimited file (same file name as the XML batch file but with a CSV extension). Some boards/schools generate these csv files to perform data analysis on the information sent to OnSIS.

 

Purge and pre-populate ALL non-enrolled student info: Y/N field (secondary schools only). If set to yes, it will include a command in the batch file that instructs OnSIS to purge ALL of the following data in OnSIS for non-enrolled students, prior to processing the contents of the file:

 

  • SHSM Program/Certification/Course info

  • Diploma info

  • Option sheets

 

If re-submitting batch files containing any of the above info, this should be set to yes.

 

Non-enrolled students are those students who require info to be sent to OnSIS but the student is not actually reported to ONSIS in the student school enrolment file. This is typically students who have completed credit requirements in a previous submission and now that they have completed their community involvement hours, their diploma must be reported to OnSIS (via the Other info option). This may also include option sheets (course selections) for students who are coming next year (via the Other info option), or students who have SHSM info (program, certification, course info) but the student is not enrolled on a count day (entered and left the school between count dates).   

 

Adult day school – Exclude attendance totals: Y/N field (secondary schools only). Report card attendance data is reported to OnSIS for publicly funded secondary schools. If this school is considered an adult day school, then set this option to Y. If this is set to Yes, the following information is omitted from the OnSIS batch file: Total classes, Classes Missed, Times Late, IEP and ESL/ELD info. Note: most adult schools are configured as continuing education schools, getting a different OnSIS submission period and batch file options.

 

For classes with zero occupancy, check adds/drops to see if these classes were populated on the count date:  Y/N field (secondary schools only). Setting this to Yes would be the most accurate option, but it does significantly increase the time to create the file. If set to No, the file will automatically skip over any classes that have zero occupancy today (which may be after the count date). This makes the generation of the file faster, but should only be used if the school is certain that they have no empty classes that may be inadvertently omitted from the file.

Student school enrolment batch file  

Default file name: E2_######_StuSclEnrol_yyyymmdd_hhmm.xml (elementary)  or S3_######_StuSclEnrol_yyyymmdd_hhmm.xml (secondary)

 

The student school enrolment batch file contains information about the students enrolled in the school on the count date. OnSIS collects student mobility info, which is the info regarding a student’s entry and departure from the school. Student mobility is with respect to what the student did since the previous submission to OnSIS. With each submission period, a student can only have 1 entry and 1 exit date and reason. When a student requires more than 1 entry or exit, only 1 can be reported and the other is ignored and is never submitted to OnSIS. If during a submission period, a student enters, leaves, then re-enters, the batch file will contain the most recent entry, however, if the student received a high school final mark between the first entry/exit dates, then the older entry date is reported and the exit date is ignored.  

 

Secondary submission periods: if a student has a future entry date to secondary school, the student will be excluded from the batch file (student is expected to be reported in the elementary batch file). These excluded students will appear in the warning/error report.

 

This file contains the information to update the OnSIS system with the students enrolled at the school on the count date.

 

For secondary schools, if the “Entry to Secondary School date (on the Enrolment tab of the student’s record) is in the future, then the student is considered an elementary student and will not appear in the OnSIS secondary files, but will appear in the OnSIS elementary files.  For example, if a student’s entry date to secondary school is 2020/09/01, the student will appear in the elementary files, not secondary files, for the 2019/20 OnSIS submissions.

 

This file also includes diploma information for secondary students who have been granted a diploma during the submission period.

 

This file also contains information on “student mobility" which is information regarding a student’s entry and departure from your school. Student mobility is with respect to what the student did since the previous submission to OnSIS.  With each submission period, a student can only have 1 entry and 1 exit date and reason.  When a student requires more than 1 entry (or retirement), only one entry can be reported and the other is ignored and never submitted to OnSIS.  Often, for secondary schools, if a final mark has been granted, the dates must show the student as having been enrolled during the time the final mark was granted.  For example, if during a submission period, a student enters, leaves, then re-enters, you only report the most recent entry HOWEVER if the student received a mark between the first entry/leave dates, then the older entry date is reported and the exit date is never reported.

Supervised Alternative Learning (SAL)  

Default file name: E14_######_SAL_yyyymmdd_hhmm.xml (elementary)  or S14_######_SAL_yyyymmdd_hhmm.xml (secondary)

 

This batch file option applies to publicly funded schools only.   

 

Supervised Alternative Learning (SAL) may be useful for encouraging some young people of compulsory school age to continue their learning, if they are at risk of not graduating because they are not attending school regularly and if other retention strategies have not been successful. When a student is enrolled in SAL, follow these steps:

 

  • In the student’s record, go to Enrolment > Enrolment register info. Set the attendance type field to SAL.
  • In the student’s record, go to Enrolment > Supervised Alternative Learning (SAL). Record the student’s SAL info.   

 

The fields collected by OnSIS are detailed below.  

 

The Ministry's of Education's description of Supervised Alternative Learning for Excused Pupils (SAL) is:

 

The parent(s) of an eligible student can apply for the student’s release from regular, full-time school attendance so that the student can participate in an alternative learning experience that is considered suitable for him or her by a committee established in accordance with the regulation governing the Supervised Alternative Learning for Excused Pupils (SAL) program. To be eligible for a SAL program, a student must be fourteen or fifteen years of age and enrolled in a secondary school program leading to a graduation diploma or certificate. Students who are enrolled in programs in board-sponsored schools, such as schools for the arts and academies, are also eligible for a SAL program. However, students in Provincial and Demonstration Schools, in care and treatment facilities, or those taking correspondence or distance education courses offered by the Independent Learning Centre are not eligible. A SAL program must be approved by the committee established in accordance with the regulation on SAL. When the necessary approvals have been obtained, the student will continue to be registered at the school until he or she reaches sixteen years of age, when school attendance is no longer compulsory. Regular contact with the student will be maintained by a teacher or other staff member, and the principal will report to parents whenever the school issues achievement reports. The school will maintain the Ontario Student Record for each student involved in the program. A SAL program may involve attendance at some classes or a full-time program away from the school. It may include one or more of the following elements: – full-time or part-time employment at an approved work placement – a program in life skills – continuing studies or other activities that the school board committee deems appropriate to the student’s needs and interests A student’s learning in a SAL program may be eligible for credit towards a graduation diploma.

 

SAL students must be given the Attendance Type SAL (on the Enrolment tab of student's record).  The student's FTE in the student school enrolment batch file will be determined based on the student's SAL attendance type. Students with a SAL attendance type will have an FTE of 1.00 if the SAL attendance type is 70 or more minutes of classroom instruction, or FTE of 0.50 if less than 70  minutes.

 

All SAL specific fields are located in the student's record (Student record > Enrolment > Supervised Alternative Learning (SAL)).

Student class enrolment / other credits

 

Default file name: S4_######_StuClsEnrol_yyyymmdd_hhmm.xml  

 

This batch file option only applies to the secondary school submission periods.

 

Only those classes which were included in the secondary class info batch files are considered.  Although classes may be excluded for other reasons, if the class is not taught during this submission period, it is excluded (e.g. October batch file will exclude the semester 2 classes, the March file includes both semester 1 and 2 classes, and the June file excludes semester 1 classes).

 

OnSIS is collecting class enrolment information for the students enrolled in secondary school classes, along with information regarding any credits attempted/earned during the academic school year.  Failed courses in the students' transcript are collected by OnSIS, along with withdrawn courses that are entered in the transcript as per full disclosure requirements.  Courses that are pre-populated from a previous submission, that have since been dropped from the students’ timetables, are also reported to OnSIS.  

 

The setup of the school and the student’s timetables needs to be accurate for this batch file. Check the following:

 

  • Calendar start and end dates of each semester/term are accurate (Setup > School > Calendar).
  • Classes have accurate schedules (semester/term/day/period info) and start/end dates (Setup > Class info > Classes).
  • Students timetables are accurate (Student record > Class info > Timetable). The timetable should include all of the student’s classes. The date the student started each class, and if applicable, the date they withdrew from a class should be accurate. Hint: From the student’s record, use the Class info > Classes option to see the student’s start/end dates for each class.  If they need editing, from the timetable option, use the More icon to access the Register option (register of add/drop dates).
  • Students should have a graduation requirement assigned (Student record > Achievement > Graduation requirements). This is used to determine whether a completed course is compulsory for graduation.

 

At the end of the semester, the students' transcripts (completed courses) must be updated with the final marks and credits earned (see Add completed courses for info on this process). The info in the student’s completed courses (Student record > Achievement > Completed courses) is where the final marks and earned credit information is recorded for use in this file.

 

This batch file also includes ‘other credits’, an area within OnSIS that allows a school to report credits earned, when the course is not possible to be reported through the student class enrolment.   Other credits are those courses entered directly into the student’s completed courses (transcript) without adding the corresponding class into the student's timetable (e.g. conservatory music credits). Courses earned through the PLAR process are not considered other credits (they are reported in the PLAR batch file). See Class list batch file (secondary) for more info on courses being reported as other credits if the course is not dual credit but the teacher’s role = U.

Prior Learning Assessment & Recognition (PLAR) credits

Default file name: S8_######_PLAR_yyyymmdd_hhmm.xml

 

 Prior Learning Assessment and Recognition (PLAR) is the formal evaluation and credit-granting process whereby students may obtain credits towards their secondary school diploma for prior learning.  PLAR applies to regular day school students and mature students who are enrolled in publicly funded secondary schools and inspected private schools that choose to implement PLAR. All boards must develop and implement policies and procedures related to the challenge and equivalency processes that are consistent with provincial policy.

 

The "challenge" process is the process whereby students' prior learning is assessed for the purpose of granting credit for a course developed from a provincial curriculum policy document.

 

The "equivalency" process is the process of assessing credentials from other jurisdictions.

 

All credits granted through the PLAR process are reported to OnSIS in the PLAR batch file. The courses in the student’s completed courses (transcript) that were earned during the submission period, that have the PLAR type field set to Challenge or Equivalency will be considered PLAR courses.

Special education

 

Default file name: E3_######_SpEd_yyyymmdd_hhmm.xml (elementary) or S5_######_SpEd_yyyymmdd_hhmm.xml (secondary)

 

The Special Education batch file contains information on the students flagged as Special Education in the Student School Enrolment batch file.  Based on the info in the student’s record using the Special education option, a student will be included in the file if the following info is applicable for the student during the submission period:

 

  • special education status is Active (on the File details card)

  • placement setting is one of the following: fully self-contained, indirect service, partially integrated, resource assistance, withdrawal assistance (on the Placement card)

  • the placement date, if entered, must be on or before the submission count date (e.g. can’t be in the future).

 

Students may have one or more exceptionality (on the Exceptionalities card). If a student has no exceptionality, he/she is considered non-identified.

 

Note:  if a student has an Active status, but no placement, the warning report will indicate that the student is excluded from the batch file.

Second language program

There are slight variations for determining students enrolled in second language programs based on the different submission periods.

 

Publicly funded elementary submissions: Based on the info in the student’s record using Enrolment > Student enrolment > Enrolment reporting (card), a student will be included in the file if the following info is applicable for the student during the submission period:

 

  • English as a second language (ESL) field = Yes; OR

  • English literacy development (ELD) field = Yes; OR

  • French/Anglais program has a value and # minutes/day; OR

  • Indigenous program has a value and # minutes/day.

 

Publicly funded secondary submissions: Based on the info in the student’s record, a student will appear in the file if:

 

  • During the submission period, the student’s timetable contains a course starting with FSF, FSG, FSI, FEF, FEG, FIF, FIG, LN; OR

  • English as a second language (ESL) field = Yes; OR

  • English literacy development (ELD) field = Yes; OR

  • (English schools) French/Anglais program has a value and student's timetable contains a non-French course where the language of instruction is French. *

 

Note: only the second language program type is identified (not minutes per day).

 

* Secondary schools are still required to track the second language program as students may not be taking a French language course (course code begins with an F) but are taking other courses taught in French (i.e. course code does not begin with an F). It will use the second language program to determine what program to send for students in this scenario.

 

Private Secondary Schools: Based on the info in the student’s record, a student will appear in the file if during the submission period, the student’s timetable contains:

 

  • A French course (course code starts with F); OR

  • A course where the language of instruction is French; OR

  • A course where the language of instruction is any native language.
