Overview

Most elementary schools use homerooms to represent a group of students who receive classroom instruction for many subjects from the same teacher. In high schools, homerooms are optional but are still used by many schools. They provide a specific time and classroom where students meet under the supervision of a teacher for the administration of school business (announcements, distribution of notes, etc).

 

Each school has a unique table of homerooms. Each homeroom has a room number, a designation (a meaningful way to identify the homeroom), and the ability to assign  up to 2 teachers.  After set up, each student can be optionally assigned to a homeroom. If homerooms change between semesters, the student’s homeroom is reassigned after the completion of a semester (e.g. the student has only 1 homeroom for the school year).

 

After the initial set up, from year to year, this table may not require changes. This setup area is used under the following circumstances:

 

  • Homerooms need to be add/edited to reflect new teachers
  • Homerooms need to be add/edited to reflect new and room assignments
  • An additional teacher is assigned to the homeroom
  • You have been recommended to use class associations to assist in creating timetables for elementary students.
  • You have been asked to review the report card terms/topics assigned to homerooms.

 

The Student tools area of Maplewood connectEd has options to help schools with the assignment of homerooms to the students. Homerooms must be defined correctly before moving onto the assign features. In most cases, the homerooms are setup first in the table of homerooms.

 

Choose Setup, then Class setup.

  

This will take you to the Class setup section where you will find the Homerooms card.

 

 

Select either the THIS YEAR or NEXT YEAR button on the card to access the homerooms that are available for the appropriate school year. The homerooms may be different for each of the school years to allow the school to plan for future changes.

 

Note: The initialize scheduling process automatically copies this year’s homerooms over to next year’s homerooms, while the year-end roll over process (transfer to current) copies next year’s homerooms over to this year’s homerooms.

Each homeroom has the following fields:

 

Room number:  The physical room in which the students in the homeroom meet. The room must first exist in the school’s table of rooms (see Rooms for more information).

 

Designation: A short code (4 character maximum) to describe the homeroom. Sometimes it is the same as the room number, other times it represents the grade or grades of the students and/or their teacher (e.g. 1A, 2A, 1SM, 3/4  for a split 3 &4, 6A, 6B).

 

Teacher(s): The teacher who is responsible for the students in the homeroom. Only those staff members with the “Is teacher” field set to Yes may be homeroom teachers. Up to 2 teachers may be assigned to a homeroom at a time.

 

Early childhood educator(s): Ontario elementary schools may select the ECEs assigned to this homeroom.

 

Class type: Ontario elementary schools must select the most appropriate type from the list (Regular, Special education, Intensive Core French, Other).

 

Achievement profile: There may be some report card set ups that have report card terms and topics assigned to homerooms rather than classes. For those setups, the achievement profile is assigned here.

Editing a homeroom

Select the homeroom from the list to access the Edit option.

 

 

Then proceed to change the homeroom information as needed. While editing a room, it is possible to navigate to the next or previous room in the list using the NEXT ROOM and PREVIOUS ROOM buttons at the bottom of page. Selecting these buttons will SAVE the changes and move to the next homeroom.

 

 

 

 

Note: Selecting a homeroom from the list provides access to the Details option as well as the Edit option. The Details option is standard on all tables, and in this case, provides nothing that isn't already displayed on the list.

   

When a homeroom is edited, all students that are currently assigned to that homeroom will have the edits automatically applied. For example, if homeroom 9SM 101 Ms. Jane Smith were edited to 10SM 102 Ms. Jane Smith, then all the students that were in 9SM 101 would be changed to 10SM 102 Ms. Jane Smith.

Removing a homeroom

As a safeguard, homerooms may not be removed if they are assigned to any students for the selected school year (this or next year). If a homeroom is to be removed, select the homeroom from the list to access the Edit option. From the edit homeroom tool bar, select the Delete icon.

 

 

Assigned classes

The ASSIGNED CLASSES tab is available for elementary schools only. This feature is beneficial when all the students in the homeroom have the same classes in their timetables. Timetables for elementary students are optional, however, most report card setups will use classes to represent the subjects on a report card. This feature allows classes to be associated with a homeroom so that timetables may be automatically created for students as they are assigned to the homeroom. Assigned classes are available for setup for both this year and next year.

 

For each homeroom, the grade levels of the students in the homeroom are specified (e .g. a split grade 1/2 homeroom is shown in the example below). For each grade in the homeroom, the potential classes for that grade are displayed and those classes associated with that homeroom are flagged with a check mark (e.g. ART1b, DAN1b in the example below). Hint: Click the header above the check boxes to sort the table by this column.

 

 

 

Group classes that share achievement topics (e.g. achievement profiles):  This option appears if the homeroom has 2 or more grades. Check this box, if the classes that have the same achievement profile are automatically grouped (so they have the same teacher, room, and class schedule). E.g. DAN1b and DAN2b would be grouped if they both have the same achievement profile.  (This might be beneficial to Ontario schools when reporting educator subject assignments to OnSIS).

Understanding auto assign feature to add classes to timetables of students in homeroom

For each checked class, pay close attention to the Auto Assign setting (either Yes or No). If the Auto Assign column indicates Yes for the associated class, it will be automatically added to the student’s timetable. If Auto Assign is set to No, the class is associated for reference purposes only but is never automatically added to the timetable. For example, some may choose to define the class associations for all the potential classes that may apply to the students in the homeroom, but only those classes that everyone gets will have the Auto Assign set to Yes. For example, the core subjects might be set to Yes, while students split into optional classes set to No. Use the drop down arrow beside the Yes or No to change this setting.

 

During the homeroom setup, when this homeroom is saved (e.g. selecting DONE or NEXT HOMEROOM or PREVIOUS HOMEROOM), the Auto Assign feature will automatically add any associated classes that are set to Auto Assign. It will only add missing classes not already in the student’s timetable – it will not remove any classes already assigned to the student that are not associated to the homeroom. MAKE SURE YOU HAVE THE CORRECT SECTION LETTER OF THE COURSES CHECKED OFF FOR THE HOMEROOM otherwise you may end up with duplicate classes in the students’ timetables.

 

Auto Assign is also used when a student’s homeroom is first specified. All classes for that homeroom/grade are added to the student’s timetable automatically (i.e. only classes set to Yes for Auto Assign are added).

 

Auto assign is also used when a student’s homeroom is changed. Those classes associated with the student’s original homeroom/grade will be automatically dropped (regardless of whether Auto Assign is Yes or No). Then, the classes associated with the student’s new homeroom/grade are added to the student’s timetable automatically, if the class is set to Yes for Auto Assign. Note: if the student has a class in their timetable that is not associated with the original homeroom, that class will not be dropped.

 

Report card marks and comments are linked to classes. If a student changes homerooms, it will attempt to move the marks and comments from one class to the corresponding new class that has the same terms and topics. If report card information cannot be moved, it will be retained with the deleted class.

Important notes before using this feature

  1. Please follow all steps and read all instructions. This new feature allows classes to be added to the classes table, and the Auto Assign feature impacts students’ and teachers’ timetables.
  2. Schools that have already assigned timetables to students may optionally follow these steps. This is a powerful feature that will save elementary schools time in the future, and therefore, we suggest that everyone consider setting this information up at some point.
  3. Note: The auto assign feature DOES NOT consider timetable conflicts when assigning students these classes automatically. Conflicts exist when more than one class is scheduled at the same time. If periods are assigned to classes, make sure that the classes associated with the homeroom do not conflict.

 

Step 1: Determine whether add/drops are to be recorded

As classes are added and dropped from a student’s timetable, the dates when the classes were added and/or dropped are automatically recorded. Every class in the student’s timetable must have a corresponding “add” record. These add/drop dates are important for secondary schools but are not as important for elementary schools.

 

If a student has been in a class since they started school this year, the add date is typically the first day of school. If timetables are being created later in the school year, it is possible to have these dates still default the first day of school for the student. There is a setting that allows the school to set the number of days into the school year before the program records the actual add/drop date (rather than just defaulting the first day of school). Check that the school’s configuration for recording add/drop dates is correct. There are 2 settings to check, and they are both found under Setup> School > Configuration.

 

  1. Days before recording timetable changes  will default 1, which means that the actual date the student added or dropped the class will be recorded starting after the first day of school. If there are 200 school days, and 200 is entered here, the actual date that the student’s timetable was change will never be recorded (it will always be the student’s start of the school year). Elementary schools may want to set this to 200. Some may want to start recording add/drop dates after the first couple of months of school when they know that timetables are in the process of being inputted (e.g. 60 days).

 

 

  1. Timetable changes apply to all semesters (only appears if school has 2 or more semesters), if set to Yes, and the school has multiple semesters, means that timetable add/drops will not be recorded until after the number of days specified into the 2nd, or 3rd, etc semester. For example, if a secondary school has 2 semesters, and 5 days is specified for “Number of Days Before Recording Timetable Changes”, then semester 2 timetable changes for the first 5 days in second semester will recorded in the add/drop area as if the student was in the class from the start of the semester (adds will be created, drops will be removed). Changes from the 6th day on will use the actual date to record all class adds and drops.

 

Step 2: Specify the Grade(s) of the Students in Each Homeroom

The next step is to specify the grade levels of the students in each homeroom. To do this, edit the homeroom, select the ASSIGNED CLASSES tab, then select the Add Grade button. It will present all the grades used by the school, from which you will select each grade appropriate for the homeroom. With the selected grades checked off, select SAVE.

 

 

You will notice that a card is created for each of the selected grades.

 

Each Grade’s card will have all of the classes associated with the selected grade (based on the grade field in the courses table). There may be classes listed (if already setup in the classes table) or there may be no classes (if they will be setup as part of the next step).

 

In the example above, the section ‘a’ grade 8 classes are already in the classes table and therefore, they automatically appear. If this list includes the classes to be assigned to this homeroom, follow step 3, option i. If this homeroom’s classes are not listed here, follow step 3, option ii (which would be the situation for this example).

 

Step 3: Select classes associated to the homeroom

Depending on whether or not the school has already setup classes in the Classes table, there are two approaches to associating classes to the students in the homeroom.

i. Classes already exist

Find the class (course code and section letter appropriate for students in this homeroom/grade), and click the box to the left of the class. Hint: use the paging controls to show more records. Change the “Auto Assign” column to Yes or No depending upon whether the class is to be automatically added as described earlier.

 

 

 

ii. Classes do not already exist

If the classes for the students in the homeroom/grade do not already exist, use the Add New Section icon in the tool bar of the Grade X – Class associations card. It will present all the courses for that grade level where you can select which courses require class sections to be created. Courses with a check mark will have a new class added to the classes table upon selecting the SAVE button shown below.

 

 

Classes will be created using the next available section letter and they will be the assigned classes for this homeroom.

 

The room, teacher, occupancy, auto assign, and language fields may be edited, if required, by selecting the field to be changed. Change the “Auto Assign” column to Yes or No depending upon whether the class is to be automatically added as described earlier.

 

For schools that actually define where classes appear in the timetable, the newly added classes will still require  edits to indicate when the classes are taught (using Setup> Class info > Classes).

 

Note: for both option i and ii above, when you select SAVE, if the homeroom has any classes set as Yes to “Auto assign”, a message will briefly appear indicating 'Students’ timetables have been checked. Any missing classes have been added.
