Student records > Achievement > Completed courses (Alberta)

 Alberta

Alberta Education is responsible for generating a student’s official transcript based on detailed course enrolment, mark and exam registration (when applicable) information that is provided by the schools to PASI. edsembli’s completed course area provides both a record of the student’s completed courses that may include courses that are not part of the official transcript, or may represent course and exam activities as they pertained to the school year and is where schools can add completed courses to be sent to PASI.

 

Alberta Education has defined a list of official course codes for courses in grade 10 to 12 that may be used on official transcripts. The official course codes are recorded in the PASI code field. PASI collects information only for these codes, which will be referred to as provincially defined codes. If a code is provincially defined, and the division has Course Mark Integration enabled, many fields are not editable, (such as course title and grade level).

 

With Course Mark Integration, provincially defined courses from a student’s timetable, including newly added classes after the start of the school year as well as dropped classes are sent in real-time to PASI. In edsembli SIS, the courses in the student’s timetable are automatically moved into the completed courses area when the school runs a process after final marks are entered (Achievement > Completed courses > Add completed courses). This process will also submit the student’s school mark to PASI.

Adding completed courses in Alberta is typically reserved for the addition of a grade 10 to 12 course that was taken outside of Alberta’s learning systems and the school is awarding an evaluated mark. These evaluated courses are also reported to PASI, and they will be considered an evaluated mark based on the evaluation method field.

 

With Course Mark Integration with PASI, provincially defined courses cannot be modified with respect to the course details that were reported based on the student’s enrolment in a class. The mark can be edited for the current school year only, and editions are reported to PASI.

The completed courses area also contains class attendance information that is automatically transferred based on the daily attendance taken by the school. This information is not collected by PASI and therefore can be edited at any time.

 

Within edsembli SIS, schools can track the compulsory credits for graduation for each completed course. As this information is calculated by PASI and the information recorded in this area of edsembli is not sent to PASI, it is also editable.

The following fields are available for an Alberta student’s completed courses:

 Completed course

Date: Year and month of completion are recorded.

Course, Title, Course grade, PASI code: Indicates the course the student completed. The title, grade and PASI code are all attributes of the course and are automatically set based on the course.

School mark: The school mark value for the course.

Exam mark: The diploma exam mark value for the course.

Final mark: The final mark value assigned to a student for a course. PASI will accept mark values between 0 to 100 or one of the following code values:

A = A letter grade

B = B letter grade

C - C letter grade

D = D letter grade (Valid until 1995/1996 school year)

F = Fail

P = Pass

Credits attempted: Indicates the credits of the course that would be applied to the student if successfully completed the course. This value is based on the credits of the class that was in the student’s timetable.

Credits earned: Indicates the credits that the student has earned.

School completed in: Indicates the school where the course was taken. Only schools in the table of schools are available for selection.

Evaluation method : Indicates the method used to determine the student’s final mark. This field is only reported to PASI when the student has an evaluated mark. An evaluated mark is a mark assigned to a student for a course that was taken outside Alberta’s learning systems. If the course was not taken outside of the Alberta learning system, it will appear as “N/A – AB defined course”. If an Evaluated Mark is to be entered, schools must select a value from the Evaluation Method Table.

Course enrolment status : Indicates the status of a course enrolment. Note: this field is tracked by PASI as part of the data for a student’s course enrolment, and is updated automatically in PASI when the student drops a class or completes the class. When it is posted to the transcript, it will have the value Completed.

Funded: Identifies that the course enrolment should be considered during funding related processes. This is a yes/no field and will be set to Yes automatically when the course is automatically added by the school at the end of the course.

External credential: Identifies that the course enrolment should be considered for eligibility for credentials not offered by Alberta Education (e.g. the Green Certificate). This is a Yes/No field.  

Course details

The fields in the course details area will be automatically populated based on the information defined for the class that was in the student’s timetable. For Provincially Defined Courses, with PASI Course Mark Integration enabled, this information may not be modified for the student.

Language of instruction : Indicates the language used when providing instruction for a class section.

Primary delivery method : Identifies how the course was delivered based on the presentation method of the largest component of instructional material.

Section delivery schedule : Indicates how the class section was delivered.

Course evaluation method : Indicates the type of evaluation associated with a course.

Course sequence series : Indicates the order in which related courses should be taken.

CTS module: Yes/no field to indicate whether the course is a CTS module. If set to Yes, 2 additional fields are enabled, which are only applicable to CTS modules:

Course instructional level : Indicates the rigor of the curriculum and thus the level of skills taught within a CTS course.

Course strand : Indicates the strand (or career area) to which a CTS course is related.

  
Locally developed course: Yes/no field to indicate that the course is locally developed course code.

Include in GPA calculation: Yes/no field to indicate whether this course is to be included in any GPA calculations that the school may want to use for their own purposes.

Course attendance

Attendance information is automatically recorded when the course is automatically added. Attendance information will only be available if the school is recording class attendance (rather than AM/PM attendance).
  

Absent: The number of times the student was absent.

Excused: The number of times the student was excused.

Late: The number of times the student was late.

Possible: The number of times the class was taught and the student was expected to attend.

Compulsory credits for graduation requirements

PASI has the student’s official transcript and is responsible for the official determination of whether a student has fulfilled all the necessary credits for graduation.
  
edsembli SIS also offers the ability to track graduation requirements, including the ability to consider credits not yet completed. If using edsembli’s graduation requirements, each completed course will have a category that is used to automatically determine whether the course is compulsory. For most situations, this area does not get changed from the defaults. See Graduation requirements for more information on how this information is used.
  

Category: This will default the category assigned to the course code. This is the category that should be used but it may be changed for this student if required. In Alberta, the category is based on the value in the ‘course subject(s)’ field of the code.

Main category group / Automatically determined: In some graduation requirements, a course may fulfil credit requirement in more than one requirement area. This area is typically not changed from the defaulted category group and yes to Automatically determined. Below is an example to show how these settings are used:

A student may be required to fulfil these 2 requirements as part of their diploma (fictitious):

- 1 credit in French or Native language or international language(main category group called Another Language).

- 1 credit in a French course (main category group called French)

The student’s course has the French category and it could therefore fulfil the requirements of either of the 2 possible main category groups. If the Automatically determined field is set to Yes, each modification to the student’s completed courses (e.g. new courses added) will give the program the ability to set the main category group to the group that best fits the student’s needs. If it is set to No, you can select the specific category group. Yes is the recommended setting.
