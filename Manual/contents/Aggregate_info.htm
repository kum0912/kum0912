Aggregate info

Aggregate data is a summary of information by category. For example, the number of children in each grade, by age or by gender. Specific, identifying information for each child is not collected. Private elementary students and students attending non-inspected private secondary schools are not issued OENs from the Ministry of Education, and therefore, OnSIS is only able to collect aggregate student data. The data is collected once a year, on October 31st.

The OnSIS batch files containing aggregate info are available using the Aggregate info option as described here.  This card will only appear for schools configured as non-inspected private schools.

This option is used to create batch files, using the CREATE button. The batch file is uploaded to OnSIS through the OnSIS web site.  Once OnSIS processes the batch file, a results file is created. The results file is downloaded from OnSIS and is then processed using the PROCESS RESULTS button.  

 

This section describes the Student info batch files. See the OnSIS Important concepts section for info on Updating data in OnSIS using the ‘Create/Upload/Process’ cycle. When the file is processed by OnSIS, typically there would not be any errors and therefore the PROCESS RESULTS functionality is generally not needed (e.g. any errors should be easily identifiable via the Ministry's application without the added steps of working with the results files in the edsembli SIS).

 

Select the batch file(s) to be created and then select the CREATE BATCH  FILES option in the bottom footer of the application.  A PDF report is also created.  If the numbers in the PDF report are accurate, submit the file to OnSIS so that the numbers are automatically recorded in OnSIS (or manually enter the numbers directly into the OnSIS site).

Enrolment, transfers, retirements and admission

When the October submission period is opened, the 'Enrolment, Transfers, Retirements and Admissions' section on the OnSIS website will be pre-populated with the enrolment at the end of the previous year.  This number cannot be changed as it represents the elementary enrolment on October 31 of the previous year).  The batch file is sending the transfers, retirements and admissions since November 1 last year until October 31 of this year.  Last year's number, minus the transfers and retirements, plus the admissions, will mathematically determine the enrolment on October 31 of this year. E.g. Enrolment October 31 last year - transfers - retirements + admissions = enrolment October 31 this year.
