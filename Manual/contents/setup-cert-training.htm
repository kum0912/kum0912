Setup > Graduation requirements > Certificates, training & awards

Overview

The certificates, training & awards table lists all the possible values that may be recorded in the certificates, awards & training section of each student’s record as shown here: 

 

 

Ontario secondary students enrolled in Specialist High Skills Major (SHSM) programs have specific certificates and training requirements. The SHSM requirements use these certificates as well, so it is important that these values are set up accurately. Students in SHSM programs will see all their certificates, training & award entries in that area of the student records, but in the SHSM requirements area, they will see a card for just the SHSM certificates & training entries that fulfil requirement for their specific program.  

 

All jurisdictions get access to this area, and all schools (elementary & secondary). Each value in the table has the following fields: 

  • OnSIS code 
  • Old Ministry code 
  • Board code 
  • Title (English and French) 
  • Hours of instruction (hidden if value is an ‘award’ (hrs = 0), shows if value is a ‘certificate or training’ (hr >0) – hours print on the SHSM record and are reported to OnSIS 
  • Effective date 
  • Expiry date 

 

Ontario secondary schools only will see an additional column in the table called ‘Used in SHSM requirements’ which will list the short SHSM program names that include this value as one of the requirements. 

 

When editing a value that is ‘Used in SHSM requirements’, there will be a sub header called “SHSM programs using this certificate/training”. It will list the long title of the SHSM programs (1 program per line). 

 

Some of the certificates and training are actual courses (these are typically titles that contain capital letters – i.e. Hike Ontario or Leave No Trace), but others are more of a category (typically titles that are all lower case – i.e. fall protection or customer service). 

 

For example, customer service can be fulfilled by:

  • A++ 
  • Customer Service Excellence 
  • Best Customer Service Practices 
  • Or a variety of other customer service courses

 

Here a school would add the actual name of the course, and associate the OnSIS code for ‘customer service’ to that code, however in order to uniquely identify the entry in the database, it must have a different MIN# or Board#.

 

Certificates, training & awards are centrally defined by the board/division to ensure accurate diploma names and Ministry reporting.   

 

 

Clicking GO will bring the user to the following interface. 

Selecting any record in the table (e.g. row click) will present 2 options: Edit and Details.   

 

*NOTE: if logged into a school, Edit option is NOT available. See the section “Row click – Editing table records & Details” for further instructions. 

 

 

CLOSE button on the bottom bar will return the user to the setup page (landing page with all the cards).

 

 

Adding Table Records - Board/Division ONLY

If logged in from the school, no add icon is shown. 

 

If logged in from the board/school, the Add icon appears in card tool bar if user has edit rights. Users with view only rights do not get the Add icon. 

 

Click Add icon. Standard Add interface will appear with fields in the following order (field names are bold – do not include #s or colon), and the card title is called Add certificate/training/award.

 

1. English/French: Default screen language. This controls whether the user is entering the English or French version of a field that has unique values for each language. Fields with underlines have unique values for each language. 
 
2. Certificate/training/award title*: alphanumeric field. Certificate/training/award title in English or French must be >0 characters in order to save. 
 
3. OnSIS code: alphanumeric field, max 6 characters
 
4. Old Ministry code: numeric field, max character count = 3 
 
5. Board code: alphanumeric field, max character count = 3 
 
6. Radio button options (below) – default “Certificate or training”

 

 Certificate or training      Award 

7. Hours of instruction:  numeric field
 
8. Effective date: date field – cannot be after expiry date 
 
9. Expiry date: date field – cannot be before expiry date


 

SAVE and ADD buttons are disabled until the required fields exist.  

 

SAVE button is enabled when all required fields are entered. Selecting SAVE will save the new entry and return the user to the table. 

 

ADD button is enabled when all required fields are entered. Selecting ADD will save the new entry and then open a new (empty) Add interface (so user can add another record to the table). 

 

CANCEL button is always enabled. Selecting CANCEL will not save the new entry. 

 

Selecting the back arrow on the main app bar should prompt the user to save new entry IF ALL required fields are entered.

 

 

Row Click - Editing Table Records & Details Option

From the table, clicking any row with table records (row click) will give 2 options: Edit (make it first option) and Details.

 

 

From a school, row click will present the Details option only.  Edit option is not available for schools.  

 

Selecting the Details option will open a standard Details form.

 

 

CLOSE or X will close the details form and return to the table (at the same page where user was).

 

Selecting the Edit option will open a standard edit form. 

 

On the Edit form, show the DELETE icon in the app bar. 

 

The following fields will be visible, just like the Add form:

 

1. English/French: Default screen language. This controls whether the user is entering the English or French version of a field that has unique values for each language. Fields with underlines have unique values for each language. 
 
2. Certificate/training/award title*: alphanumeric field. Certificate/training/award title in English or French must be >0 characters in order to save. 
 
3. OnSIS code: alphanumeric field, max 6 characters
 
4. Old Ministry code: numeric field, max character count = 3 
 
5. Board code: alphanumeric field, max character count = 3 
 
6. Radio button options (below) – default “Certificate or training”

 

 Certificate or training      Award 

 

7. Hours of instruction:  numeric field
 
8. Effective date: date field – cannot be after expiry date 
 
9. Expiry date: date field – cannot be before expiry date

 

PREVIOUS (if there is a previous row in the table) and NEXT (if there is a next row in the table) buttons will appear.   

 

Selecting PREVIOUS button will save the edits and then reopen the edit form with the previous record (row) in the table. Selecting NEXT button will save the edits and then reopen the edit form with the next record (row) in the table. 

 

 

Selecting the back arrow on the main app bar should prompt the user to save edits.  Standard prompt should appear:

 

 

Deleting Table Records

Editing a table record (row click > edit option), will show the delete icon. This icon will be disabled if the role has been assigned to any staff. 

 

DISCARD button will remove the record from the table. CANCEL will retain the record in the table. 

 

 

Table Tool Bar - Search

Table Tool Bar - Change View Style (List/Grid)

Standard list/grid view icon appears in the grid. Default grid view but allow user to change to List view.
