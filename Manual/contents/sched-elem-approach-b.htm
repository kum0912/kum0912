Approach B:  Classes are created using the traditional setup of classes

 

With this approach, schools can use the familiar set up classes to create all the classes for next school year.  Then the 'assign students to classes' tools can quickly add the classes to the student's timetables.  With this approach, student timetables are never automatically updated to reflect the classes that are assigned to the students in the homeroom. As new students are added to the school and they are assigned to a homeroom, the student's timetable will be empty until the classes are added.

 

B1. Prepare for next year

The scheduling process beings with running the ‘initialize scheduling’ process.  This process duplicates the school information from this year, into the next year.  This makes re-entering last year's homerooms, advisor groups, courses, classes, and semester/term/day/period information unnecessary if they will be similar for the next year.  It also advances the students’ next year grades.

 

  • Overview
  • Initialize next year
  • Sem/term/day/period

 

B2. Setup classes

A unique class should exist for each unique group of students that meet for instruction in a subject taught by a teacher. The classes should be adjusted to match next year's needs using Scheduling next year > Classes > Classes. This may include:

 

  • Adding new classes
  • Removing classes no longer needed
  • Changing class teachers
  • Changing class rooms
  • Adjustments to the class schedule (when it's taught)

 

Often the actual schedule of when the subjects are taught is not formally recorded in the SIS, allowing the teachers to teach subjects based on their own schedules.  Sometimes, classes are formally scheduled to avoid teacher and room conflicts (e.g. physical education, French, music). When classes are formally scheduled, the schedule is determined manually by the school and then is entered into the SIS.

 

  • Classes not formally scheduled into periods may be assigned a floater pattern (a user-defined day/period pattern that has no periods selected).
  • Classes scheduled into periods may be assigned to semester/term/day/period patterns, but often the schedule is random and creating unique patterns would be time-consuming.  In the set up of the class, it may be flagged as requiring a special schedule which then allows the user to select the specific semesters/terms and days/periods unique to that class.

 

If new rooms are added to the school next year, or rooms are no longer available, adjust the table of rooms using Scheduling next year > Classes > Rooms.

 

Once the classes are set up, see an overall view of the teacher and room timetables using the  Teacher master timetable and Room master timetable cards.

 

 

 

B3. Setup homerooms

Elementary students are typically assigned into a homeroom and the homeroom teacher is typically the teacher responsible for the core subjects. The homeroom should be adjusted to match next year's needs using Scheduling next year > Homerooms > Homerooms. This may include:

 

  • Adding new homerooms
  • Removing homerooms no longer needed
  • Editing homerooom teacher(s)
  • Editing room and designation info

 

B4. Assign students to homerooms

Once the homerooms are set up correctly, the students should be assigned into their next year's homerooms. There are many different options to assist in assigning the student's next year's homerooms. These options are more convenient than editing each student, one at a time, and there should be one method that works well for your school.  Descriptions of each option is provided on the card.

If printed lists of the students in a homeroom are needed, use the Homeroom lists card to generate those reports.

 

 

B5. Assign students to classes

Classes are typically added to students based on their homeroom, and tools are available to add classes to the students in a specific homeroom (or any other grouping of students (e.g. grade)). Note: these tools only impact the group of students at the time the tool is run. New students will need to have classes added directly in their timetable, or by running these tools again.

 

  • Add classes
  • Delete classes
  • Clear timetables

 

B6. Advisor groups

If advisor groups are used, they should be adjusted to match next year's needs using Scheduling next year > Advisor groups > Advisor groups. Advisor groups are similar to homerooms where a room and teacher(s) is/are assigned to the advisor group. The name of the group can be anything the school wants. Some schools use advisor groups to indicate the guidance councilor assigned to the student. Some schools have students meet regularly in groups where the students meet with an advisory teacher to discuss general school related topics.

 

If advisor groups change from year to year, there are many different options to assist in assigning the student's next year's advisor groups. These options are more convenient than editing each student, one at a time, and there should be one method that works well for your school.  Descriptions of each option is provided on the card.

If printed lists of the students in an advisor group are needed, use the Advisor group lists card to generate those reports.

 

B7. Print timetables

Printing timetables for students, teachers, rooms is often not necessary in elementary schools, but the option is available if needed. Class lists are also available.

 

 

B8. Year-end roll over

The board/division/district is responsible for the actual Transfer to current process, however schools should make sure that they have run the Archive attendance & achievement process. This process may be done at any time during the scheduling process, as long as it is run after the current school year is done and no further attendance or report card data is being entered.  

 

 

Typically the roll over is done over the summer, before the students start school again in the fall. It cannot be run before the current school year is finished (e.g. it cannot be done in June), nor can it be run when the school still needs the current year's info for reporting and other processes.

 

The transfer to current process takes all the 'next year' info and copies it into 'this year'. This info includes:

  • courses
  • classes
  • rooms
  • timetables
  • students' grades, homerooms and advisor groups

 

Student who are no longer returning to the school become past students, and all other students are assigned the school year status of 'this year and next year'.

 

Once the school has transferred to current, there should be no further use of the Scheduling next year options.  All changes to data need to be done to the 'this year' info.
