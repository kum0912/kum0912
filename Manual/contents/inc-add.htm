Add incident

When you add an incident from the main menu you will need to select the student. When you add an incident from the student records screen you will be adding the incident directly to the record of the student involved.

 

To add an incident choose Incident > Student incidents and select GO from the Add incident card.

 

The Add incident screen will appear:

  

Enter incident details

Enter incident details.

Note: Before you can select students, you'll have to populate the required location field. Choose from the drop down list of fields that were created under Setup > Incidents > Locations.

 

Date: Enter or select the date the incident occurred.

Location (mandatory in Ontario only): Windows shows “Other” but we should show “None selected” if empty.

Time: Enter or select the time, it is required in Ontario only.

Time frame: This is optional (can be set to None-selected.

Reported by: Select from the drop down list from the Person types you set up using Setup > Incidents > Person types. It will also include an option “Other” that is not in the setup table. This is an optional field.

How it works:

* If ‘reported by’ is a person type table entry, then look at the ‘role’ defined in the table to filter the “Name” field to show just those staff members with that role.  (e.g. if selecting reported by Teacher, and teacher is defined as role T, then show the names of all the staff with role T.

* if ‘reported by’ is ‘Other’ (e.g. not a table entry), then show 4 additional fields:  Title, Given name, Surname, Phone number.

 

 Students involved

Next select the students involved by using the Add icon. The standard Search student form will appear.

Check the boxes beside the applicable students.

Hint: If more than one student is involved, checking the box, and performing another search will keep the original student selection in memory. Make your selections and click SELECT.

 Edit student incident details

Once you have selected all of the students involved it's time to enter incident details. Row click and select Edit.

 

 If you need to remove a student from the incident, select the Delete icon. You will be asked for confirmation.

 

The Edit student form will appear.

Details has the following fields:

Student incident note: There is no character limit.

Parental contact: Y/N check box (default all Y/N fields as unchecked during add process).  If Y, then show Contact date and Contact time.  Default today’s date and time.  These fields are optional and so there should be some way to have no date or time (confirm with Rick that they can be empty).

Weapon involved: Y/N check box  (if Y, show Type of weapon (drop down field – show title and onsis code); if N, hide Type of weapon).

Police involved: Y/N check box (if Y, show Officer name (alphanumeric field) and Badge number (alphanumeric field; if N, then hide fields).

 

Student details (at time of incident) has the following fields:

Grade:  Default the student’s grade (this year). Field is editable (select from school’s table of grades).  Note:  Windows allows non-grades to be entered but then gives error upon save.

Room number: Default the room portion of the student’s homeroom (this year).  Alphanumeric field can be edited (4 chr max).

Designation: Default the designation portion of the student’s homeroom (this year). Alphanumeric field can be edited (4 chr max).

Teacher: Default the teacher portion of the student’s homeroom (this year).  If more than one teacher is assigned to the homeroom, use the Teacher (not additional teacher field).  This field is editable (select from school’s list of staff members).

Date of first incident letter:  Default nothing (can be left blank).

 Infractions

 Add an infraction

Infraction details (sub-header) has the Infractions card which lists the infractions (what the student did wrong). Once you have added an infraction, Consequence, Suspension/expulsion program, and Appeal/outcome cards will be available.

 

To add an infraction use the Add icon on the Add infraction card.

Infraction: Select an Infraction from the drop down list you set up under Setup > Incidents > Infractions.

Primary infraction: If this is the first or Primary infraction, then the Primary infraction check box will be checked.

Violent incident: Default checked or unchecked based on default in Infraction table (e.g. selecting infraction ‘Using a weapon to cause or to threaten bodily harm to another person (E02)’ should default Yes

The student’s primary infraction is listed first.  Subsequent infractions can be listed in any order (order of entry would be best if possible).

 

 Edit an infraction

To edit an Infraction simply row click and the Edit infraction card will appear.

To delete an infraction simply select the Delete icon. You will be asked for confirmation.

Consequence

The Consequences card lists the consequences (what happens to the student as a result of their infractions). The Consequence card will not be visible until you have added an infraction to the Infractions card.

Add a consequence

To add a consequence use the Add icon on the Consequence card.

 

The following fields are on the add/edit consequence interface:

Consequence: Select from the drop down field of options you set up under Setup > Incidents > Consequences. The values should include both the title of the consequence (French title if using in French and title if it exists) AND the ministry code e.g. Suspension (4).

Primary consequence: Yes/no check box.  

  • student can have only 1 primary consequence and they must have 1 flagged as primary consequence.
  • * If adding the first consequence for the student, it should default Primary consequence = Y.  
  • * If adding a second or subsequent consequence for the student, it should default N.  If it is changed from N to Y, then it must change the original primary consequence to No.  
  • * if adding the first consequence for the student, it should default the consequence defined for the primary infraction (e.g. infraction table has a default consequence for each infraction).

Imposed by:  Drop down options from the list of Person types that you set up under Setup > Incidents > Person types, however it will only include those values that have the field ‘Able to suspend/expel’ set to Yes.  (there is no “Other” option like in the ‘reported by’ field).  E.g. here are some examples of the person type table showing this field:

* If ‘Imposed by’ has a role (example 2 above), then look at the ‘role’ defined in the table to filter the “Name” field to show just those staff members with that role.  (e.g. if selecting reported by Teacher, and teacher is defined as role T, then show the names of all the staff with role T).

* if ‘Imposed by’ has no role (example 1 above), then show 4 additional fields:  Title, Given name, Surname, Phone number.

*‘Imposed by’ field is optional outside Ontario. Ontario has these business rules:

(Ontario only) – ‘Imposed by’ is required if consequence is a suspension or expulsion (based on consequence table where ‘suspension type’ is anything other than ‘none’ as explained in R24)

(Ontario only) – if consequence type is expelled (code 4 or 5), Imposed by must be the board (code 1).  (Technical note using onsis file terminology:  INFRACTION_OUTCOME_TYPE = 4 or 5 must have INFRACTION_AUTHORITY_TYPE = 1)

 

Search: You can add a staff member not listed by performing a Staff search.

Phone number: Enter the contact phone number for the Imposed by person.

Resolved:  Y/N field (default unchecked)

Resolved date: date field – enabled if Resolved = Y, hide if Resolved = N

Suspension/expulsion details (sub-header): hide if the consequence assigned to student is setup where the Suspension type is none (see example below).  Show if the suspension type is anything other than ‘none’ – e.g. one of the 4 suspension expulsion /mandatory elective options.  Here is an example of the fields when setting up consequences in the table.

 

Suspension/expulsion details

Effective date:  date field.  Default today’s date.  Date cannot be before the incident date.  Must be before the end date.  If date is outside school calendar, set ‘Automatically calculated ...’ to No and disable it (auto calculation cannot function for any dates outside of the school year). Toast message “Duration cannot be automatically calculated for dates outside the school year”.

End date:  date field.  Cannot be before the effective date.  Field may be automatically calculated if ‘Automatically calculate weekday duration’ = Y. If date is outside school calendar, set ‘Automatically calculated ...’ to No and disable it (auto calculation cannot function for any dates outside of the school year).  Toast message “Duration cannot be automatically calculated for dates outside the school year”.

Duration:  numeric field (can be up to 3 characters max, 0 – 999, or #.## such as 2.25).  Field is automatically calculated if ‘Automatically calculate weekday duration’ = Y and both effective and end dates are recorded.

Automatically calculate weekday duration:  Y/N field.  If set to Yes, entering either the end date or the duration will automatically calculate the other field.  For example, if an end date is entered, the duration is automatically calculated.  If the duration is entered, the end date is automatically calculated.  Field is automatically set to No if effective or end dates are outside school calendar.  

Include exam days:  Y/N to indicate whether exam days are counted in the calculation of the duration or end date.  This field is enabled when ‘automatically calculate weekday duration’ is set to Yes.

 Suspension/expulsion program

The Suspension/expulsion program card details information about the program applied to the student. This section only appears for Ontario.

This section only appears if the student has a consequence that is a suspension or expulsion (based on consequence table where ‘suspension type’ is anything other than ‘none’

Card will have an ADD icon if no program assigned to the student, an Edit icon if info exists.  

Default collapse card if no program assigned to the student.  If expanded, and no program, show empty state message such as “Student has no program. Use the add icon to add the student’s program info.”

  • If a Suspension/Expulsion Program is reported, then Program Status must also be entered
  • If a Suspension/Expulsion Program = ‘Both Academic and Non-academic’, then a Non-academic Program must be selected (NOTE: you can select more than one value)
  • If a Suspension/Expulsion Program = ‘Academic only’, then a Non-academic Program must be disabled/hidden.
  • If a student is suspended for less than 11 days and greater than or equal to 6 days, you must have a Suspension/Expulsion Program type indicated (‘Academic only’ or ‘Both Academic and Non-academic’)  
  • If a student is suspended for more than 11 days or is expelled then only 'Academic and Non-Academic' program should be available in the drop down.
  • Students suspended for 5 days or less can enroll in a suspension/expulsion program but is not mandatory; therefore, no warning message will be generated.
  • If a student has multiple values assigned for ‘Non-academic program(s)’, list them together (e.g. value1, value2).

 

 Add suspension/expulsion program

To add a program select the edit icon. The add suspension/expulsion program form will appear.

 

Specify details by using these fields:

Suspension/expulsion program:  drop down field with the following values:

- Not assigned   (rename to ‘None selected’ if easy)

- Academic only

- Both academic and non-academic  

- Non-academic only

Default “None selected”.  If user changes from a program to ‘none selected’, clear out the values in non-academic program and program status fields.

Non-academic program(s):  drop down field – allow multi-select of values.  

-Disable field if program=‘not assigned’ OR ‘Academic only’.  

-At least one value is required if program = ‘Both academic & non academic’  OR ‘Non-academic only’ (make field mandatory in this situation).

 

Program status:  drop down field (only one single value can be selected).  

-Disable field if program=‘not assigned’. Required if program ‘Academic only’ OR ‘Both academic & non academic’ OR ‘Non-academic only’ (make field mandatory in this situation).

Note:  There is no limit to characters in this optional field with no limit.

Select SAVE when you have finished adding fields.

 

 Edit a suspension/expulsion program

 Appeal/outcome

This section only appears if the student has a consequence that is a suspension or expulsion (based on consequence table where ‘suspension type’ is anything other than ‘none’.

Card will have an ADD icon if no appeal/outcome info assigned to the student (Appeal initiated = N), an Edit icon if info exists (Appeal intiated = Y).  

 

 Add appeal/outcome

To add an appeal/outcome select the edit icon.

Specify details by using these fields:

Appeal initiated: Y/N check box. If N, then disable all fields (clear out values if data previously entered).

Appeal date: date field. Mandatory. Must be after incident date.

Decision date: date field. Optional. Must be after appeal date.

Outcome: drop down field. Optional.

New duration: 3 character field (e.g. 0 to 999 or #.## like 2.25).  Field enabled if outcome = ‘Reduced suspension’ OR ‘Reduced – expelled for school.

Select SAVE when you have finished adding fields.

 

 Edit appeal/outcome

To edit an appeal/outcome simply select the Edit icon and the Edit appeal/outcome card will appear.

Select SAVE when you have finished adding fields.
