Scheduling next year > Year-end rollover - Overview

Before the start of the new school year, the school needs to close off the last year and roll-over to the new year. The close off last school year activities include:

  • Running the option Scheduling next year > Year-end rollover > Archive Attendance & Achievement. This may be done anytime after the previous school year is finished. This option may be run multiple times, with each time only adding changes since the previous time it was run. If it is run in early summer, and again just before transfer to current, the second time it is run will be much faster since it will only need to update differences since the first time it was run.
  • Generating all necessary lists and reports based on the previous year's data (e.g. the data in 'this year'), including updating the student's completed courses (transcripts) and report cards.
  • Submitting all data submissions to requesting agencies (e.g. Ministry provincial reporting, federal reporting, post secondary reporting).
  • Completing all scheduling activities to the point that you can continue any additional activities in 'this year' (e.g. Scheduling next year menu options are not needed any more).

Before rolling over to the new school year, the board/division/district must perform the following activities:

  • Use Scheduling next year > Year-end rollover > Manage year-end backups to define schools with similar structures that can roll-over at the same time.
  • Use Scheduling next year > Year-end rollover > Clear online registration data to clear the staging table data for all schools (since all of the registrations should have been processed by now).

Once the school has closed off the last school year, the board/district/division will be responsible for running the Transfer to new school year process. Transfer to the new year, performs many activities. Here are some examples:

  • creates a year-end backup of the data  
  • copies the student's next year grade, homeroom, advisor group and timetable into the student's this year grade, homeroom, advisor group and timetable (and similarly updates the teachers' homerooms, advisor groups and timetables). Creates internal transfers to show the date the student advanced grades.
  • changes the school year status for students/staff as follows:
    'This year only' status changes to 'Past'.
    'Withdrawn' status changes to 'Past'.
    'Next year only' status changes to 'This year and next year'
  • copies the following setup tables from next year to this year: courses, classes, rooms, homerooms, advisor groups.
  • sets up the marks and comments based upon the achievement profiles for classes and homerooms.
  • sets the edsembli record owner to the appropriate school for each student.
  • Updates locker availability for students who have left the school.
  • Applies course fees where set up.

While a school is in the process of transferring to current, all access to the school's data will be prevented.

 

Once the school has been moved to the new school year, the following activities should be followed:

  • The Scheduling next year option should not be accessed, except for viewing information, if needed.
  • Changes will be done using 'this year' data, not 'next year' data. This includes changes to grades, homerooms, advisor groups, timetables, room, courses, classes. This includes the setup tables and the student/staff data.
  • Edit the school calendar to ensure that holidays are defined. Confirm that start/end dates and semester/term dates are correct. Carefully check that the day in the cycle is accurate, and if the school has students who attend on alternate days, the alternate days must also be checked. Use Setup > School > Calendar to do this.
