Determining when a student is expected to attend school

Every student in a school may have daily attendance recorded for each day that the student is expected to show up to school. Typically you would enter attendance for the current date, but you can enter attendance for any date in the current school year, past or future, as long as the student is enrolled in school on that date.

 

A student’s entry date to this school and exit/retirement date, if applicable, are criteria considered when determining whether the student is expected to be at school. If a student has multiple entry and exit/retirement dates, these dates are also considered.

 

For students with the ‘by period’ attendance method, the timetables must exist because the attendance is recorded for the classes the student is taking in each period on each date. Class add and drop dates are also considered to determine accurately recorded class attendance.

 

The school’s calendar defines which days are non-school days and no attendance would be recorded. Sometimes, a regular school day becomes a non-school day due to bad weather or some mechanical issue forces the closure of the school. If all students in the school are affected by the closure, see Unexpected school closure.

By period vs morning/afternoon attendance

Each student has an attendance method field that determines how many times a student’s daily attendance is recorded. The attendance method field is located in the Grade info card available from the Class info, Classes menu in the student’s record.

 

Elementary students typically have morning/afternoon attendance recorded. Secondary students typically have attendance recorded for each period where the student is expected to show up to a class. A school with kindergarten to grade 12 students may require both attendance methods. See Setup - Parameters for information on how to set up the method(s) required by your school and how to set the default attendance method for each grade in the school.

 

There are 2 possible attendance methods:

1. By period

With this method, attendance is entered for each class in the student’s timetable.  

 

If a class is not assigned a specific period, or if it is not taught today, then you cannot record attendance. If a student has an alternate day schedule (e.g. the student does not attend school every day), attendance cannot be recorded on the days where the student does not attend.

 

If your school has classes that span multiple periods, be sure to check that your attendance parameters are set up to reflect whether you want to enter attendance only once for these classes or if you want to enter attendance for each period. When a student is absent for a multi-period class, the parameters let you determine if the absent is counted as 1 day or is counted for each of the periods (e.g. count as 2 if student missed a double period class). See Setup - Parameters for details.

2. Morning/Afternoon

With this method, attendance is entered twice a day, corresponding to the morning and the afternoon attendance:

 

 

Students with the morning/afternoon attendance method will also need to ensure that the attendance type field is also set accurately. The attendance type will indicate if a student is attending half time AM or PM only, or full time, or full days on alternate days, etc. For example, a Half Time (AM) attendance type will mean that only morning attendance may be recorded for the student. Both the attendance method and attendance type fields are in the Grade info card available from the Class info, Classes menu in the student’s record.

 

 

If a school has students attending full days on alternate days (e.g. half time kindergarten student comes Monday, Wednesday and every other Friday), make sure the school’s calendar has the alternate day schedule defined correctly. See Setup – Calendar for more information.

Attendance codes and reasons

Attendance is recorded using a list of attendance codes and reasons that may be defined centrally by the board/division or by the school. The attendance code A is reserved for Absent, and P is reserved for Present, but all other letters of the alphabet are user defined. An unlimited number of reasons may be associated with each code to further classify the attendance code. For example, a school might have a L – Late attendance code with 3 reasons: L – Late – Missed bus, L – Late – Slept in, L – Late - Other. Reasons are optional as some schools prefer to enter the reason in the student’s attendance notes.

 

If a student is absent and you know the reason, then record attendance using the appropriate code and not just Absent. For example, if the student is absent due to an appointment, rather than recording A - Absent, the attendance should be recorded using an alternate attendance code such as M - Medical appointment, or M - Medical appointment – dentist (if reasons are defined).

 

We recommend that reasons are not defined for A – Absent so that this code may be used to quickly identify the students with unexplained absences.

Sometimes, the attendance information needs to be summarized in a manner other than based on the A to Z attendance codes entered for the students. For example, you may just want to know how many times the student was absent from school considering all the different codes. Each attendance code has 2 settings that help summarize attendance information.

 

  1. Every code has an official setting that indicates whether it is considered Absent, Present, Late, Suspended, or Grant day (e.g. unexpected school closure).
  2. Every code also has a classroom setting that indicates whether the student is expected to show up in the classroom. If the student is not expected to show up to the classroom, there are 2 options: absent (student is absent from school) or excused (student is excused from class because they are elsewhere attending a school related activity).

Unexplained absence

If a student’s attendance is recorded as Absent (e.g. no specific attendance code was entered), it is considered an unexplained absence.  Depending on the age of the student and the school’s attendance practices, these students are listed for quick follow up. See Call home for more information.

 

Some schools actively follow up with students to determine the reason for the absence, changing the attendance code from absent to a truant code when the student has no valid reason for missing school.

Entering attendance

The bulk of attendance entries is based on the attendance that is taken in the classroom for a group of students in a homeroom or class. The teachers record this attendance directly into edsembli CONNECT from the classrooms where possible, otherwise, the attendance information is recorded on paper for office staff to enter.

 

The office typically enters attendance on an individual student basis as attendance information becomes available from phone calls, notes, and students who show up at the school office when arriving late to school.

 

The school office can enter attendance for groups of students with known attendance such as field trips, sport teams, and bus delays. Excursion groups may be created to record unique groups of students (e.g. girls volleyball team, junior band, football team) so that attendance may be quickly assigned to the students in the excursion group. In addition to excursion groups, attendance may be entered for these other groups: all students, students in a homeroom, students in a grade, students in a class, students with a specific defined character, and students on a bus.

Note: School office staff have the ability to lock the attendance information they entered so that a teacher cannot change it. Although this should not be necessary, the feature is available if required.

Attendance notes

In addition to recording attendance, attendance notes may be recorded each day noting additional information such as sign in/out times, and phone call conversations while attempting to find out more information about an unexplained absence. Attendance notes appear on all attendance entry forms, and they may be accessed in the Notes and Attendance areas of a student’s record.

Attendance info in student records

All attendance information recorded for each student is accessible from the Attendance menu in the student’s record.

Unexpected school closure (e.g. weather)

If an unexpected school closure happens at the start of the day and all students in the school are affected, the school’s calendar may be edited to change the date from a normal school day to a non-school day. We suggest the changing the day type to a Grant day (sometimes referred to as “G” day or General day). To do this, go to Setup, Calendar. Select the date and then edit the date to access the day type field.

 

If an unexpected school closure affects only some students or affects only part of the school day, do not change the calendar. Instead, use the Group/Excursion attendance entry method to assign the appropriate attendance code to the affected students and time period affected.

Holy day (religious holiday)  

If your school has students that miss regular school days due to religious holidays, be sure to create an attendance code that will accurately reflect how these days are to be reported in your province. Typically, the attendance code is set up as Absent for the official setting, while the Class absent/excused setting can be anything. Ontario schools have specific regulations that require these absences to appear as General (‘G’ day), and therefore they should be set up as Grant for the official setting.

Cancelled class/not enrolled

When entering a student’s attendance, in addition to the attendance codes and reasons available to the school, you may notice the following additional attendance entries:

 

c- Cancelled class: This code may be used in the ‘by period’ attendance method to indicate that the  class was cancelled. The code will be treated the same as P – Present, and will therefore only appear on views where the school attendance codes are displayed. It will be treated as present when viewing attendance using the official or class absent/excused code definitions.

 

^  - Not enrolled: The student is not enrolled at this school. This code may be used when a student has certain days when they do not attend this school (e.g. a full time student goes to another school every Wednesday for an alternate program). This code will appear as * when viewing the attendance using school attendance codes, N when viewing the official codes (N = Non instructional day). Note: using this code will reduce the number of possible times that a student could attend the class or am/pm time period.

Sign in/sign out

When students arrive late to school, or leave the school early, most schools require the student to go to the office to sign in or sign out so that the office is aware of the student’s attendance.

 

For students with the morning/afternoon method of attendance, the student’s attendance should be edited to reflect what is happening with the student. If your school uses the morning/afternoon method only, you may skip the rest of this section.

 

For students with the ‘by period’ method of attendance, the student’s attendance includes sign in and sign out options for each period as shown here:

 

 

The exact time the office signs a student in or out can be automatically added to today’s attendance note if the check box “Apply sign in/out information to the note” is checked. This setting will be retained for all students until it is unchecked. Here are examples of the notes that will appear when this setting is checked and the office checks the Out box for period 2 and the In box for period 5:

 

Signed out:  10:09 AM (Period 2 )

Signed in:  2:12 PM (Period 5)

 

Sign in: When a student is absent  from school, and then returns back to school, the student may be signed in by checking the box under the ‘In’ column for the appropriate period.  If a student signs in for period 2, they are expected to go to their period 2 class.  The student’s attendance will change to present for period 2 and all subsequent periods, however, if the teacher has already recorded attendance for the class, that attendance will not change automatically but can be edited if appropriate.  For example, if a school records sign in at the end of the day, if a student signed in for period 2 but did not go to their period 3 class, it will detect that the teacher recorded the student as absent and will not overwrite this attendance.

 

Sign out: When the student is attending school but needs to leave the school early, he/she may be signed out by checking the box under the ‘Out’ column for the appropriate period. If a student signs out for period 3, he/she is not expected to go to their period 3 class. If you want a student to be considered present for period 3 because part of the class was attended, check the sign out box for period 4 (the first period in which the student will be missing for the whole class). Using the period 3 sign out as an example, the office should change the period 3 attendance from Absent to a code that represents the reason the student is leaving school. This new code will appear for all subsequent periods automatically, however, if the teacher has already recorded attendance for the class, the attendance will not change automatically but can be edited if appropriate.

 

If a student signs in and out (or vice versa) in the same period, the 2 actions will cancel each other out (both boxes cannot be checked). Recording the sign in/out in the notes will provide details outlining the student’s activities for that period.

Notifications

This section applies to the ‘by period’ attendance method only. This feature is planned for a future release.

 

Sometimes notifications are created to ensure that student attendance accurately reflects what happened in the classroom. When a student signs in to the office, the office will assume that the student attends class and the attendance reflects this assumption.

 

If the sign in occurs after the teacher has taken attendance, the teacher will automatically have a new notification created to ensure that the student actually went to class after signing in. The teacher will need to confirm the student’s attendance at some point (later in the day or the next day) to clear the notification. Here is the notification that the teacher will see:

 

 

As soon as the teacher selects one of the options for the notification, it will be removed from the teacher’s list of notifications. If the teacher selects the STUDENT DID NOT SHOW UP, the student’s attendance for that class will change to Absent (unexplained absence) and the student will appear in the list of students with an unexplained absence.
