Fees > Student fees

 

 

Fees are school-specific, however, with permissions, you may see the fees recorded by another school. The school creates a list of fee types to broadly describe the type of fee. Examples might include course fee, bus fee, student activity fee, lock purchase. When you apply a fee to a student or group of students you can add comments to help clarify the fee. As fee payments are made to the student’s account, you can easily track how much the student still owes. You can also set up deposits for borrowed items such as text book. Deposit funds that are collected at the start of the course can then be returned after the student returns the text book.

 

When a fee (charge) is added to a student’s record, it will appear on Account Bills and other similar reports as a C with their respective invoice number.

 

Add a fee

When a fee (charge) is added to a student’s record, it will appear on Account Bills and other similar reports as a C with their respective invoice number. You can add a fee using these methods:

 

• Individual student (using two access points)

1. Choose Fees > Student fees and select GO on the Individual card.

 

 

Choose the student from the standard select student form.

 

2. Choose Student records and select the student. Then select Fees from the submenu.

 

Select the Add fee icon from the FAB on the student record SUMMARY tab:

 

  

 

 

To add group fees see Group fees for more information.

 

Add a fee main screen

When you add a fee this is the form you will use to add information. If you are adding a fee to an individual student you won't see the Select by area. This is only relevant when adding a fee to a group of students.

 

    

These are the Add fee fields:

 

Amount:  The dollar amount of the fee in the format ##.## (e.g. enter 12.50 for a $12.50 fee, or enter 12 for a $12.00 fee, there is no need to enter the dollar sign). Entries must be a positive number.

 

Date:  The date of the transaction (it will default today’s date but may be changed to any date as long as it falls within the range of 1 year before or after today’s date).

 

Fee type:  Select the type from the list of possible types defined by your school.  Example types might be Student activity fee, Yearbook, Lock purchase, etc.  See Setup fee types for additional information.

 

Invoice number:  The invoice number is automatically assigned, but if needed, it can be changed to any alphanumeric value as long as it is a unique number.  The application will automatically default the next available invoice number based on the starting invoice number set up for the school.  See Setup ….

 

Payment due date:  Enter the date the payment is due.  This field only appears if the school is set up to record due dates (see Setup ….).  If the payment is not received by this date, the fee will be included in the “Overdue unpaid fees”. The date can be any date after the date of the transaction.

 

Applied taxes:  If the fee is taxable, select the tax to be applied from the list of tax types (e.g. GST).  It will default the taxes associated with the fee type, but it may be removed if appropriate.  This field only appears if the school has defined at least one tax type as part of the set up (where the tax rate is also defined).  See Setup ….

 

Total taxes:  If a tax type was selected in the Applied taxes field, the dollar amount of the taxes will appear here.  This field is not editable as it is automatically calculated based on the tax rate (e.g. if a tax is applied with a rate of .05 (5%), a $10 fee will show $0.50 for the total taxes). This field only appears if tax types have been set up.

 

Comment bank: You will see the comment bank only if the school has set up a list of comments (see Setup ….)).  The comment bank lets you select from a list of predefined comments. To use this feature, select the drop down arrow to select the desired comment, then select the Copy icon to place the comment in the comment field.

 

Comments:  You can record a comment when you add a fee.  

 

 

Dialogue messages

When you save a fee is, you may see a dialogue message relating to account limits if the school has set up either of these account limits as (see Setup … for details on setting up these limits):

 

  • A school can set up an account limit so that a student’s outstanding (unpaid) fees cannot exceed a set maximum. If an account limit is set up, and if the addition of a new fee will exceed that amount, you will see this dialogue message:

 

This student has an account balance that exceeds the school’s account limit.  Do you wish to override this limit?  CANCEL     YES, ADD NEW FEE

 

  • A school can set up a fee type limit so that a student’s outstanding fees for that type cannot exceed a set maximum. For example, a school may set up a student activity fee type with an upper limit of $50.00.  When adding a new student activity fee type with a fee greater than $50, a dialogue will be presented as follows :

This fee amount is greater than the limit set by the school.  Do you wish to override this limit?  CANCEL     YES, ADD NEW FEE

 

You will also see this dialogue if a fee amount plus the outstanding fees for this type exceed the upper limit (e.g. new student activity fee is $25 and student already has outstanding student activity fees of $30).

 

When adding a fee, if the student has payment credits, the user will be prompted asking if they would like to apply the credits. See Payment credits for more details.

 

Edit fees

Once a fee is assigned to a student, you can make changes, but editing the fee has some rules:

 

  • You can make edits to the date of the transaction, the payment due date (if applicable), and comments.
  • All fee, or tax type edits need a comment to explain the reason for the amount change. 

    Technical notes:  The invoice number cannot be edited, but it will automatically include ‘A-‘ before the number (e.g. A-1112) to indicate that the fee amount has been adjusted.  In most places, the new amount will appear with the original invoice number, however, in the database, it will record the original entry followed by transactions with the adjustments (e.g. if the original fee was $25 and then it was changed to $20, the database has 2 transactions:  the original invoice 1112 with $25 and A-1112 with ($5.00).  Some reports will show this as the 2 transactions. If a fee has been edited, only the new amount ($20.00) will appear on a student’s record, not the original amount ($25.00).

 

 

 

When you edit (adjust) a charge that has a payment associated with it, you will see a warning message.
