This option is used to create batch files, using the CREATE button. The batch file is uploaded to OnSIS through the OnSIS web site. Once OnSIS processes the batch file, a results file is created. The results file is downloaded from OnSIS and is then processed using the PROCESS RESULTS button.  

 

This section describes the Class info batch files. See the OnSIS Important concepts section for info on Updating data in OnSIS using the ‘Create/Upload/Process’ cycle.

The OnSIS batch file options are detailed below. Prior to selecting the CREATE BATCH FILE option, determine whether other options on the form should be selected.   

 

Purge:  Y/N field. See Important concepts - Purge check boxes for more info.

 

Purge and pre-populate ALL OnSIS info: Y/N field. If set to yes, it will include a command in the batch file that instructs OnSIS to purge and prepopulate all data in the submission period before processing the remaining contents of the batch file. This will allow the school to restart the entire submission period over again.  Please note that this will remove any changes made in OnSIS, including educator data that may have been updated manually or submitted by another application (e.g. a HR system).   

 

Save results to CSV file (aka Also create CSV file): Y/N field. If set to Yes, the CREATE BATCH FILE option will create the requested batch file(s) and the contents of the batch file(s) will also appear in a comma delimited file (same file name as the XML batch file but with a CSV extension). Some boards/schools generate these csv files to perform data analysis on the information sent to OnSIS.

 

For classes with zero occupancy, check adds/drops to see if these classes were populated on the count date:  Y/N field (secondary schools only). Setting this to Yes would be the most accurate option, but it does significantly increase the time to create the file. If set to No, the file will automatically skip over any classes that have zero occupancy today (which may be after the count date). This makes the generation of the file faster, but should only be used if the school is certain that they have no empty classes that may be inadvertently omitted from the file.

Class list batch file (Secondary schools)

The secondary class list batch file contains a class code which is a unique name representing the group of students meeting in the same room for instruction.  The class code does not have to match any Ministry defined course code. The Ministry’s common course code will be linked to the class in the Course segment batch file. Both of these batch files may be created at the same time.

 

OnSIS’ CLASS_CODE tag will be populated with the class code (e.g. ENG1Da) used in the Classes table (Setup > Class info > Classes). There are two exceptions:

 

  • If the class is grouped, the word “–Grouped” is added after the first class in the group.  Grouped classes (sometimes also referred to as combined classes) occur when the same teacher teaches multiple subjects at the same time.  Classes will be automatically grouped even if they are not specified as grouped in the classes table (e.g. if you have 10 different classes taught at the same time by the same teacher, they will automatically be considered grouped classes).

 

  • If a value exists in the OnSIS class code field for the class, this value is used instead.  This field is typically blank but then gets populated for the semester 1 classes after the October submission is completed, and for semester 2 classes after the March submission is completed.  This field helps retain class groupings between OnSIS submissions, and it is automatically populated as part of the last step in the October and March submissions. See Populate OnSIS class codes for more information on how this field gets values.

 

Classes included/excluded

Classes are included/excluded based on how they are setup (Setup > Class info > Classes).  Some class fields are defined at the course level (Setup > Class info > Courses).  If the class’ course code has a department, the setup of the department is considered (Setup > Class info > Departments).

 

Classes are included if:

 

  • Class has a course grade = 9 or higher  

  • Class is taught during the current submission period   

  • Class is reportable  

  • Exclude from OnSIS submission field = No  

  • Course department does not have a Ministry code = 00

  • Teacher of the class has a role that does not = U (un-reportable)

  • Teacher of the class has a role that = U and the class is dual credit (in course type field)*

 

Classes are excluded if:

 

  • Class has a course grade = 8 or lower  

  • Class is not taught during the current submission period   

  • Class is not reportable

  • Exclude from OnSIS submission field = Yes

  • Course department has Ministry code = 00  

  • Class is not dual credit (in course type field) and teacher of the class has a role = U (un-reportable)**

 

* In staff records, each staff member has a role field.  The role field can be set to U (un-reportable) for any staff members that are in the school teaching who should be ignored when reporting to OnSIS. If the educator's role is set to U (Staff records> Employment) all classes taught by the educator will not be established in the OnSIS batch files (class list, course segment, student class enrolment, and educator class assignment). The exception to this rule is DUAL CREDIT classes, where these classes are required on report cards and are to be submitted to OnSIS in the class enrolment batch file.  The teacher of the dual credit class may be a contact outside of the school, so OnSIS has a class type of NE (No Educator) that may apply in these situations:

 

  • Class type will automatically be sent as NE (DC: External Educator – aka no educator) if course type = dual credit and no teacher assigned to the class

  • Class type will automatically be sent as NE if course type = dual credit and teacher assigned to the class has a role of U.

 

** Non dual credit classes that have a teacher whose role = U:  When the credits are granted to the students, they will be reported through the Other Credits area. A warning will appear on the generation of the class list, course segment, and student class enrolment batch files to identify any classes that will be ignored if the role is U. The warning is:  Class code has been excluded from file because the educator's role is U. Role U is reserved for educators that are needed in the school, but are not reported in the school's educator assignment in OnSIS. Credits earned by the students will be reported as Other Credits.

 

Credit Recovery and E-learning Classes

Credit Recovery classes should be set up in the classes file using the common course code of the course being recovered (e.g. ENG2Da or ENG2DRa), with the Course Type set to Credit Recovery.  In rare situations, a student may be recovering multiple credits in the same period in the timetable.  In this situation, the classes may not have a period assigned to them but this affects the FTE of a part-time student. Placeholder classes, starting with RCR, that are not-reportable, will need to be added to the timetable to augment the FTE accordingly.  As long as the RCR placeholder class is not reportable, it will not be sent to OnSIS, however the course being recovered will be reported.

 

E-learning classes are reported by the school offering the e-learning course even though the student's main school collects the funding for the e-learning course (and in turn pays the school offering the course).  E-learning classes offered at your school should be set up in the classes file using the common course code, with the course type set to E-learning through Provincial LMS or E-learning not through Provincial LMS (LMS = Learning Management System).  Note: if you are the main school not offering the e-learning course to one of your students, if the student is PT, you will need to augment the student's timetable with a placeholder class, starting with FTE. As long as the FTE placeholder class is not reportable, it will not be sent to OnSIS.

 

E-learning classes may not be grouped with non-e-learning classes, and if it occurs, they will be sent as distinct classes in the batch file and a warning will appear (upon creating the file) indicating that this has been done.

Course segment batch file (Secondary schools)

Each class code will have at least one segment in the Course Segment file, corresponding to the 5 character Ministry defined course code taught in that class. Where grouped classes exist, the class code will have 2 or more course segments in the Course Segment batch file. Course codes with the 5th character containing 1,2,3,4,5,6,7,8, or L are considered locally developed. If it is locally developed, the first 5 characters of the course code are used in the LOCAL_DEV_CRS tag. If it is not locally developed, the first 5 characters of the course code will appear in the MINISTRY_DFND_CRS tag.
