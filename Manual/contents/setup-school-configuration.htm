Setup > School > School configuration


As part of the setup area for every school and board/division (central office), there is a configuration area that has a number of varied options that help define how the SIS will operate in each school or from the board/division. The board will not see this card.

 

From the main menu select Setup > School. This will take you to the School section.  

 

 

Select GO on the card to open the School configuration page.

 

 

 

Options

This card displays Specialist High Skills Major information (SHSM).

 

 

 

Edit options

Select the edit icon to access the Edit options form. Users with security rights can edit.

 

 

Record homeroom internal transfers?: Select check box to indicate yes or no. Set to yes if you want an internal transfer automatically recorded When a student changes homerooms.  If internal transfers exist, it is possible to determine homeroom enrolment numbers on past dates even if the students have since changed homerooms.

Course selection approval (LATER)

Timetable changes apply to all semesters?: Select check box to indicate yes or no. If a school has 1 semester this option will not appear. (Available only if the school has multiple semesters)  Set to yes if you do not want timetable add/drops recorded until after the number of days specified into the 2nd, or 3rd, etc semester. For example, if a secondary school has 2 semesters, and 5 days is specified for “Days before recording timetable changes”, then semester 2 timetable changes for the first 5 days in second semester will recorded in the add/drop area as if the student was in the class from the start of the semester (adds will be created, drops will be removed). Changes from the 6th day on will use today’s date to record all class adds and drops.

Days before recording timetable changes: Enter a value from 1 to 365. The number of school days to wait before recording add/drop dates for timetable changes.  Enter 1 (the default value) to record the actual date the student added or dropped the class starting after the first day of school. If there are 200 school days, and 200 is entered here, the actual dates that the student’s timetable was changed will never be recorded (it will always be the start of the school year). Elementary schools may want to set this to 200. Some schools may want to start recording add/drop dates after the first couple of weeks of school when they know that timetables are in the process of being inputted (e.g. 10 days).  Note:  number of days refers to dates in the school’s calendar (e.g. typically Monday to Friday, including non-school days such as holidays and PA days).

 

As classes are added and dropped from a student’s timetable, the dates when the classes were added and/or dropped are automatically recorded. Every class in the student’s timetable must have a corresponding “add” record.  If a student has been in a class since they started school this year, the add date is typically the first day of school. These add/drop dates are important for secondary schools but are not as important for elementary schools.

Enable verification?: Select check box to indicate yes or no for address verification. Set to yes if address verification tables have been set up and the school now wants to verify student addresses against the verification tables.

 

Allow streets outside boundary?: Set to yes if a student’s address may be permitted even if it is not within this school’s boundary. Set to no if a student’s address must be within this school’s boundary.

 

Allow all addresses?: Select check box to indicate yes or no. Set to yes if a student’s address may be any address (allow any address even if it is not standardized).

 

SHSM requirements for ‘other credits’ (English, Math, Science, etc):  Schools cannot edit this field.

 

Click SAVE when you have finished making changes, or CANCEL if you change your mind.

  

Configuration

This card displays the board configuration and defaults for new continuing education students/classes settings.

 

 

Edit configuration

Select the edit icon to access the Edit configuration form. Users with security rights can edit. This sample shows setup for a Continuing education school that is operated and funded privately.

 

Here are the configuration fields; the options you see depend on your selection of School operating type and Funding type.

 

School status: Still in operation, Closed after (then enter a date), Opened after (then enter a date).

 

License type: This field will say Board, Division or District based on the license assigned to the school and is not editable. If school license type = secondary, then show ‘Combined elementary & secondary school’ field

 

Combined elementary & secondary school?: Select check box to indicate yes or no.

 

Elementary school number: Enter a numeric value.

 

School operating type: Select from the drop down list of Continuing education , CTCCF, E-learning, Regular or Summer.  If ‘School operating type’ is Regular or E-learning AND School license = Elementary AND funding type is Public,  show ‘Requires access to continuing education registers (IILE, Literacy & numeracy).  If ‘School operating type’ is Regular OR E-learning AND School license = Secondary AND  Combined elementary & secondary’ = Yes AND funding type is Public,  show ‘Requires access to continuing education registers (IILE, Literacy & numeracy).  

 

Language: Select from the drop down list. The language of the school.  English, French, and English/French (e.g. bilingual) are the possibilities .

 

Funding type: Select a Funding type from the drop down list of Native, Catholic, Public or Private.   If Funding type = Native, show ‘Native Operation’.  (If Funding type = Native, ‘Native Operation’ must have a value selected from table.) Show asterisk with field name to indicate it’s required field.

 

If funding type = Public or Catholic, hide ‘Type of School’ and ‘Affiliation/membership’.

If funding type = Native or private, show  ‘Type of School’ and ‘Affiliation/membership’.

 

Native Operation field drop down list: Sect from the drop down list of Band, Federally, Privately or Provincially operated.Optional field unless Funding type is Native.

 

Inspected secondary school?: Select check box to indicate yes or no. This field is visible only if you select Private or Native funding type.

 

Type of school: Select a school type from the drop down list. This field is visible only if you select Private or Native funding type. ‘Type of School’ and ‘Affiliation/membership’ fields are optional. If no values are selected, show “None selected” for the values.

 

 If funding type = private OR native AND license type = secondary, show “Inspected private secondary school’

 

 If school operating type = Continuing education OR Summer, show subheader ‘Defaults for continuing education students/classes’ and show the 2 fields ‘ Continuing education student type’, and  ‘Continuing eduation course offering type’.   Values in these 2 fields are optional.  If no values are selected, show “None selected” for the value. Hide this area if not continuing ed or summer school.

 

If school operating type = CCTC, show subheader ‘Default for Care, Treatment, Custody and Correctional (CCTC) students’.  This has one field:  Designate school.  Show ‘None selected’ if no value. Hide this area if not CCTC school.

 

Affiliation/membership: Select from the drop down list. This field is visible only if you select Private or Native funding type.

 

Continuing education course offering type: Select from the drop down list. This field is visible only if you select Continuing education or Summer from the School operating type.

 

Designate school:Select from the drop down list. This field is visible only if you select CTCCF School operating type.

 

Continuing education student type: Select from the drop down list. This field is visible only if you select Continuing education or Summer from the School operating type.

 

School authority type:  (Alberta only) The school’s authority type based on the recognized classifications supported by Alberta.

 

School classification type(s):  (Alberta only)  The school’s classification type based on the recognized classifications supported by Alberta Education.

 

School program offering type(s): (Alberta only) Identify the type of program(s) offered by the school.

 

Grade offering type:  (Alberta only) Identifies the type of programming offered by the school for the identified school grade offering.  A column will appear for each grade in the school’s table of grades (Setup, School setup, Grades), and a row will appear for each of the programs listed in the PASI code table for ‘School grade offering type’.

Indigenous and Northern Affairs Canada (INAC)
