Scheduling next year > Year-end roll over > Transfer to new school year

Board/district/division (central office) only.

The Transfer to new school year option transfers all the information from the 'next year' data into the 'this year' data as explained in the overview. This transfer (sometimes referred to as transfer to current), must be done by the board/district/division (aka the 'central office'). The central office must first run the transfer process, followed by each school. Once the central office has transferred, this option may also be used to monitor the transfer status of all the schools.

 

Before rolling over to the new school year, the board/division/district must perform the following activities:

  • Use Scheduling next year > Year-end rollover > Manage year-end backups to define schools with similar structures that can roll-over at the same time.
  • Use Scheduling next year > Year-end rollover > Clear online registration data to clear the staging table data for all schools (since all of the registrations should have been processed by now).

 

Prior to running Transfer to new school year, please confirm the following:

  • The schools have run Scheduling next year > Year-end rollover > Archive Attendance & Achievement.
  • The schools have completed all the necessary lists and reports based on the previous year's data (e.g. the data in 'this year'), including updating the student's completed courses (transcripts) and report cards.
  • The schools have completed submitting all data submissions to requesting agencies.
  • The schools have completed all scheduling activities to the point that they can continue any additional activities in 'this year' (e.g. Scheduling next year menu options are not needed any more).
  • The current school year must be finished (you cannot run the transfer process if the school year calendar's end date is in the future).

 

Once the school has closed off the last school year, the board/district/division will be responsible for running the Transfer to new school year process by following the steps below:

1. Board/division/district transfers to the new school year

2. Schools transfer to the new school year

 

Monitor the transfer status of all school using the TRANSFER TO CURRENT TASK QUEUE tab.

 

Notes:

All access to the schools data will be prevented during the transfer process. This includes the use of the SIS, teacher/parent/student access, database updates, third party integration, scheduled tasks and other service queue tasks.

 

If running the transfer process to move the schools from the 2018/2019 school year into the 2019/2020 school year, all active schools must have finished moving into the 2018/2019 school year first (e.g. you cannot have an active school still operating in the 2017/2018 school year - it needs to transfer to the 2018/2019 school year before commencing the 2019/2020 transfer to new year).

 

Schools which are closing are expected to run the transfer to new school year process at the end of their last year.  Schools which are re-opening after being closed need to run transfer to current before becoming active, although scheduling processes can occur first.  

 

 1. Board/division/district transfers to the new school year


From the board/division/district, select Scheduling next year > Year-end roll over > Transfer to new school year. Set the calendar start date and end date. These dates will be the default dates for all school calendars. Once the dates are set, select the START button.

Upon selecting the START button, a backup will be made based on the settings in the option Manage year-end backups.

 

 2. Schools transfer to the new school year

From the board/division/district, select Scheduling next year > Year-end roll over > Transfer to new school year. The transfer form will be distinctly different to accommodate all the setting required for transferring schools. The form has 2 distinct functions, available through these 2 tab navigation options:

TRANSFER TO CURRENT: This tab is used to transfer schools to the new school year.

 

TRANSFER TO CURRENT TASK QUEUE: This tab is used to monitor the transfer status of the schools.

 

 TRANSFER TO CURRENT tab

This tab is used to transfer school(s) to the new school year. The form has many options that are explained below.

Schools

First the school(s) to be transferred need to be selected.  Select from elementary schools, or secondary schools.  Note that the schools to be transferred must have the same setup, meaning they must be all elementary schools or secondary schools, and the selected school must have the same number of semesters/terms. The selected schools will appear in the list of schools with a check mark in the first column. Schools may be removed if they are to be excluded, but ideally all similar schools are transferred at the same time so that the year end backup contains the year-end data for all the similar schools.

 

Optionally show schools that are already transferred by selecting the 'show completed schools' option.

 

Default calendar dates

Once all schools are selected, set the Default calendar dates. The dates include the start and end dates for the school year, and the semester/term start dates based on the setup of the selected schools (e.g. if 2 semesters and 2 terms, there will be 3 start dates to enter (semester 1 term 2, semester 2 term 1, and semester 2 term 2) as the first semester/term will start on the school's start date. Make every effort to record accurate dates to minimize the effort to adjust calendars for each school.

 

Options

Carefully review the transfer options, detailed below.

Where the options is not self-explanatory, an info icon will appear so that it may be selected to show more info on what the option means.


Attendance type changing to full-time: (elementary schools only) Select the grades where student have an attendance type other than full-time, and you would like the students to be automatically changed to full-time. For example, if a school has kindergarten students that are half-time, and they when they advance to grade 1, they are full-time, select 1 for this option.

 

Grade internal transfer date: Select the date to be used as the internal transfer date for a student's grade change. August 15th is defaulted.

 

Course fee charging date: Select the date that is to be used as the date that a course fee is added to a student's fees.

 

Course fee due date: Select the date that is to be used as the due date for any course fees that are added to a student's fees.

 

Clear current students' lockers: Y/N field. If set to yes, the students will not have lockers assigned to them. If the student has a student-owned lock, the lock will remain. If the student has a school-owned lock, the lock will be remain. If the lock is built in to the locker, it will be cleared.

 

Do not clear Next Year Only students' lockers: Y/N field. If set to yes, the students who are new to the school (e.g. the students who have the school year status 'next year only') will keep the locker assigned to the student.

 

Assign default achievement profiles: Y/N field. This should be set to yes to ensure that the terms and topics are correctly assigned to the classes so that teachers may begin establishing markbooks as soon as the transfer is completed (assuming the office has given permission). If yes is selected, the new classes will be assigned the default achievement profiles. If the profiles are defined after the transfer, they may be assigned by the school using Assign achievement profiles to classes.

 

Delete existing markbook bank before recreating based on current markbooks: Y/N field. This should be set to Yes to reduce the potential teacher error when they copy the previous year's markbook as a template for the new year's classes teaching the same curriculum. When it is set to yes, the teacher is limited to selecting only from last year's markbook templates in the bank. If it is set to no, the teacher can select from all the markbook templates in the bank, which often includes outdated templates that may be easily selected by the teachers.

3. Select the START button

Select the START button in the bottom bar to start the transfer to new year process for the selected schools. Monitor the progress using the TRANSFER TO CURRENT TASK QUEUE tab.

 TRANSFER TO CURRENT TASK QUEUE tab

When schools are selected for transfer and the START button has been selected (as described above), there is one task created for each school to be transferred. Each school must be transferred completely before the next school may start its transfer.  See the current transfer status of the schools by selecting the TRANSFER TO CURRENT TASK QUEUE. This will operate like the task queue available from the main app tool bar, however it will be limited to showing only those tasks that are related to the transfer to current roll-over activity.
