Ontario > OnSIS > Sign off in OnSIS

 

If you are unsure how to sign off a submission period in OnSIS, refer to your board for assistance, or contact the OnSIS help desk. These are very general instructions to describe the process.

 

When all the school data is submitted to OnSIS, either through direct input into OnSIS, or by uploading batch files, a validation report must be run from OnSIS. If errors were found, they must be corrected before sign-off may begin and warnings should be analyzed to see if data correction is needed. Here is the validation flow chart from the OnSIS User Guide - Signing-Off a Submission.

 

 

When the validation report is error-free, the verification reports must be run in OnSIS. If there are no errors with these verification reports, and the numbers and totals are accurate, the school sign-off button appears in OnSIS and the user may initiate the sign-off process.  If no errors are found during the sign off, the user confirms school sign-off using the appropriate button and this will lock all data so that no further changes can be made as illustrated in this flow chart:

 

 

 

Here are some images to show where these options are located in OnSIS.

 

The ‘Sign-off School Submission by the School’ option is used to access the validation reports and verification reports. The sign-off button will be available when the reports are error-free:

 

 

 

 

The board will need to sign-off the school submission as well using the “Sign-off School Submission by the Board” option. The board sign off will validate business rules for students across the board and against other Ontario boards (e.g. an active student cannot have MAIN_SCHOOL_FLAG = Yes in more than one school).

 

 

The Request Data Correction option may be used to request that the Ministry re-opens a signed-off submission period.
