Overview

The Class info option in a student’s record contains information related to the courses, classes and teachers involved in the student’s education during the current school year (this year) or the upcoming school year (next year).

 

Past students do not have access to the Class info menu, however their historical class information is available from the Achievement menu.

 

The Timetable menu option is located in the Class info submenu.

 

 

The Timetable option presents the student’s classes organized by periods, days, terms and semesters. Each school’s setup defines the number of semesters, terms in a semester, days in cycle, and periods per day. Each class offered by the school is assigned to the appropriate semester(s), term(s), day(s) and period(s) that correspond to where that class is taught. This information is used to determine where the class appears in the student’s timetable. If all classes in a student’s timetable are full year (scheduled for all semesters), or if the school’s setup is 1 semester, any references to semesters will be hidden. If all classes in a student’s timetable are scheduled for all terms, or if the school’s setup is 1 term, any references to terms will be hidden. If the student’s classes do not rotate between days, all days will be combined rather than showing a timetable for day 1 and another timetable for day 2, etc.

 

You can view a student’s timetable in 3 formats:

 

  • Today
  • Summary list
  • Summary grid

 

 

See Today’s timetable, Summary grid timetable, and Summary list timetable for details on each of these timetable views. The summary grid is the defaulted view.

 

Although you may see the student’s timetable in many different formats, there are many common features shared between all the views as described in Common features of timetables.

 

Selecting the Edit icon from any of the timetable views, shows one common edit screen (class adds and drops) as described in Editing timetables. Please make sure the school’s calendar date is set to the correct date before adding and dropping classes from the student’s timetable. If the student is starting a class after it has started, this date will be used to determine when the student first attended the class. If the student is dropping a class before it is finished, the date will be used to determine the first date the student no longer showed up for class.

 

In addition to viewing timetables, you can print a copy by clicking the More icon and selecting the Print timetable option (or the Print floating action button in the bottom right corner).See Printing timetables for more information.

 

Today (Today’s timetable)

Today’s timetable view is the only timetable view available for students who have withdrawn from the school, providing the ability to see the timetable for any past date when the student would have had a timetable.

 

This view will use the school’s calendar to determine today’s date. The school’s calendar defines what semester, term and cycle day to consider for each date. The student’s classes for each period will display, and if the time is during the school day, the class the student is currently attending is highlighted:

 

 

 

You will see today's date by default, but you can change the date by using the arrow buttons located on the left and right of today’s date.

 

 

You may also select the date to access a calendar control to select a specific date.

 

The resulting timetable will consider the student’s entry and exit/retirement dates to determine if the student is enrolled on that date. If the student is not enrolled, or if the student has no classes on that date, a message will appear stating “Student has no classes on this date”. If a student’s school year status is withdrawn, this message will appear for today’s date, however, you may go back to a previous date to see the student’s timetable on a prior date.

 

Today’s timetable view will consider the dates on which a student added or dropped a class so that the timetable is a true reflection of the student’s classes on the date. See Editing timetables (adding and dropping classes) for more information.

Summary grid timetable

The Summary grid view is a traditional timetable view where the student’s classes appear in blocks in the grid corresponding to where the class is taught. This view is not available for students who have withdrawn from the school.

 

Each unique class will be colour-coded to show where the class rotates if applicable. To add or remove colour use the palette icon to toggle off and on colouring.

 

 

If the student’s classes change at the start of a new semester and/or term, then the timetable will appear on a unique card corresponding to each semester and/or term. Use the Show more and Hide all icons in the card tool bar to expand and collapse the cards.

 

It is also possible to filter the cards on the view using the drop down semester, term and day filters located above the timetable cards. For example, selecting just semester 1 from the drop down semester filter, will hide all semester 2 and above cards.

 

 

Summary list timetable

The summary list view of the student’s timetable is similar to the Summary grid view but the student’s classes are in list format with all the day 1 classes listed for each period, followed by the day 2 classes listed for each period. The list format is better suited for view a student’s timetable on smaller devices. This view is not available for students who have withdrawn from the school.

 

Just like the summary grid view, if the student’s classes change at the start of a new semester and/or term, the timetable will appear on a unique card corresponding to each semester and/or term. These cards can be expanded and collapsed.

 

 

Common features of timetables

The school determines what semester(s), term(s), day(s), and period(s) apply to each class. A class can appear in a student’s timetable as long as semester, term and day information is defined (where applicable). Some classes are assigned into ‘no period’ (e.g. period X or floater classes). Classes that are not assigned into a period will appear after the regularly scheduled classes. When editing the student’s timetable, these classes are grouped together on their own card titled Additional classes.

 

The school’s setup defines the number of semesters, terms, days, periods, and period start and end times. These attributes define the blocks in the timetable, and the student’s classes are used to populate the blocks in the timetable.

 

For each class, you'll see the following information:

 

Class: This is the class from the student’s timetable. Class (e.g. MAT9Ab) is a combination of the course code (e.g. MAT9A in all caps) and the section letter which is a lower case letter a to z (e.g. b in this example). In the school’s classes table (e.g. the master timetable), a unique section letter is used for each unique group of students that are taught the course.

 

Title: The title of the course.

 

Room: The room number where the class is taught. If a class is taught in multiple rooms, where the room changes on different days, the room will appear as M. Select the class to access the details option to see class details including specific room information.

 

Teacher: The teacher or teachers assigned to the class.

 

Each class in the timetable grid may be selected to see more details based on the setup of the class.

This Year vs Next Year

As shown below, when viewing the timetable in the 'summary list' or 'summary grid' format, a tab strip will appear with up to 2 options:

 

 

THIS YEAR will appear if the student is enrolled in ANY school in the board/division/district during the current school year. NEXT YEAR will appear if the student will be enrolled in ANY school in the board/division/district in the upcoming school year. The student’s school year status at each school determines whether or not a student is considered as attending that school for this year and/or next year. School year statuses are recorded in the student's record using Enrolment > Student enrolment, and they are interpreted as follows:

 

This year and next year: student is considered as enrolled in the school on the THIS YEAR and the NEXT YEAR tabs.

 

This year only: student is considered as enrolled in the school on the THIS YEAR tab.

 

Next year only: student is considered as enrolled in the school on the NEXT YEAR tab.

 

Withdrawn: although the student is no longer enrolled in the school, the THIS YEAR tab is available to see the student’s timetable for the days when the student was enrolled.

 

Use the tab strip to navigate between the student’s timetable for this year or next year. When both options  exist for the student, it will default THIS YEAR. When viewing the student’s timetable using the today format, it is always considering the student’s “this year” timetable and therefore the tab strip is not applicable.

Flex, CTS, and continuous entry classes

Some classes may include an icon to provide a visual indication that the class may be one of these unique class types. The icons, explained below, will provide a tool tip when the icon is hovered, and may provide additional functionality when the icon is selected as described here:

 

Flex class (Alberta only): The class is considered a flex class, which is a unique class type available for those schools offering flex periods. The flex class is scheduled into the student’s timetable and typically all students have the same flex period. Students can pick what they want to take each flex period from a wide range of offerings created by the teachers in the school (e.g. one teacher may offer extra math help, one teacher may offer badminton in the gym, and another teacher may offer a meditation class). The teacher offerings are unique for each time a flex class is held throughout the school year, and each time the student can choose something different to take. The flex class in the course workload card is really a placeholder class. The icon indicates that this class has unique flex class functionality in edsembli CONNECT for teachers and students.

CTS class (Alberta only): The class is considered the master class of a Career and Technology Studies course or a module class. Typically, the CTS master class is scheduled into the timetable in a specific period, and the student takes many modules as part of their course. The CTS module classes are not scheduled into a period. HoveriHoveriHoveriHoveriHoveriHoveriHovering over the icon will indicate whether it is a CTS master or CTS module.

Continuous entry class: The class is considered a continuous entry class, which means the student may start and complete the course at their own pace. The student will have their own start and end times.

 

Editing timetables (adding and dropping classes)

Note: If the student’s school year status is set to withdrawn, editing the timetable will not provide the functionality to add and drop classes. Instead, the Edit icon provides the functionality to edit the register. See Register automatically tracks add/drop dates.

 

Use the Edit icon in the tool bar if classes need to be added or dropped from the student’s timetable.

 

The edit timetable interface will present the timetable in a grid view, listing the student’s classes in the first column, followed by columns for each period in the first semester/term, followed by columns for the next semester/term (if applicable) as shown below:

 

 

You will see some coloured blocks, corresponding to the spots in the timetable where the student is attending classes. For each coloured block, you will see a number corresponding to the number of students in that class. Other blocks may show a letter and number to let you know that another class section of the same course is offered at that time (section letter and class enrolment are provided should you be interested in moving the student into that class). If an asterisk appears (e.g. b*29), the school has other sections of that course at that same time (if it’s not a coloured block, it will show the section with the lowest enrolment).

 

 

If the student has classes that are not assigned to any specific period or day (e.g. period X or floater classes), they will appear in their own card titled Additional Classes with an indication of the semester/terms in which the classes are taught.

 

 

There are many different ways to make changes to the student’s timetable, described in the subsequent sections.

 

The student’s timetable is not officially saved until you actually click the SAVE button. This allows you to try various changes to the student’s timetable but revert back to the original timetable if you don’t like the changes. See Saving timetable edits or reverting to original timetables for more information.

 

Selecting a class from the first column of the grid (under the Course column) will present a menu of additional options. These options, if applicable, will appear:

 

Class details: Provides additional information (the information may vary depending on whether you are selecting a class or a course. It will present the details just like you would see in the setup of the class or course (when you row click a class or course in Setup > Class info > Courses or Classes).

 

Drop: Drops the class from the student’s timetable.

 

Dual enrolment: (Alberta only) Provides a yes/no field  to flag this class as a dual enrolment (for reporting to PASI). Check the box if the student is completing this class in partnership with a post-secondary institution.

 

Class specific options: See Flex, CTS, and continuous entry classes for additional options that may appear for these class types.

Adding a class

Adding classes is typically done by typing in the class (course code and section letter if known) or by typing in as much or as little of the course code or typing in a key word from the course title (e.g. chemistry) in the box that says “Add by typing class, course or title then select from list"” as shown below.

 

 

Based on the characters you type in the box it will start building a list of potential matches as shown below:

 

 

From the selected list of matching courses and/or classes, select a class if you know exactly the class you want to add to the student’s timetable, or select the course (which is bolded) to see where it’s scheduled.

 

If a class is selected, it will add the class to the student’s timetable. The class block will be red if the newly added class conflicts with existing classes, and therefore additional changes must be made to the timetable before it may be saved.

 

If a course is selected, the course will appear after the student’s classes. Course rows are easily distinguished by the fact that they have no section letter and the row is highlighted. Class section and class size information appears in the blocks as shown below.

 

 

From the course row, select a block with a class section to add the student into that class. The course row will be replaced by a class row that is repositioned in the list so that it appears in chronological order by period, and the newly added class block will be coloured. If it is red due to a conflict, additional changes must be made to the timetable.

 

Dropping a class

Selecting a class (or course) from the first column of the grid will present a menu of additional options. Select the Drop option to remove the class (or course) from the timetable grid.

 

 

Timetable Register

When a student’s timetable is saved, any newly added or dropped classes are automatically recorded in the register. The computer time is used to record the “Last modified” date and time. The school’s calendar date is used to determine if the student is adding a class after it has started, or dropped before the class is finished. The register is only showing information for this school’s timetable activity.

 

The school’s calendar date is set to the correct date before adding and dropping classes from the student’s timetable. If the student is starting a class after it has started, this date will be used to determine when the student first attended the class. If the student is dropping a class before it is finished, this date will be used to determine the first date the student no longer showed up for class.

 

Although this is done automatically, there are times when you may want to see or edit the register add/drop dates. The register is accessed by selecting the More menu to access additional options, including the Register option as shown below:

 

  

You will see the Register option on these tool bars in the student’s record:

 

  • Class info, Classes

  • Class info, Timetable

  • Class info, Timetable, Edit timetable *

 

* Since add/drop dates are saved when the student’s timetable is saved, selecting the Register option within edit timetable will first prompt to save the timetable.

The following information is recorded in the register:

 

Last modified: This is the computer date and time when the class’ register entry was last automatically modified. This field is not editable.

 

Class: The class being referenced. This field is not editable.

 

Class enrolment date: This is the first day of class for the student. The class start date in the school’s classes table will appear unless the student added the class after the class started.

Class end date: This is the last day of class for the student. This field is not editable as it automatically populated base on the class end date in the school’s classes table, or it is the school day before the drop date.

 

Drop date: When the class was dropped from the timetable, the school’s calendar date was automatically used to record this date. It represents the first day the student did not attend the class.

 

Semester/term: This is the semester/term pattern assigned to the class. Pattern names are school defined but should be meaningful enough to provide information as to when the class is/was taught.

 

Day/period: This is the day/period pattern assigned to the class.

 

Edit Register

Select a register entry (e.g. row click) to edit the entry, if needed. The date the student started the class (enrol date) may be entered (it must be on or after the class start date). The drop date is only available for classes that have been dropped from the student's timetable. The drop date must be before the class end date. The end date field will automatically adjust to the day before the drop date (the last date the student attended classes).

 

 

Alberta schools: Instructional minutes received is an optional field, however, if the school wants to request funding for a dropped class, this field must be populated (to help support the request for funding). This field exists for dropped classes only.

Delete register entry

Register entries should only be deleted if the student never attended a class. Use the delete icon (located in the tool bar when you edit the register entry). Since it is not possible to add register entries (without adding the class back into the student's timetable), caution is advised when the delete action is used.

Editing student’s homeroom

Many schools have a correlation between a class in the student’s timetable, and the student’s homeroom. For example, if the student’s period 1 class is in room 200 with Mr. Smith, the student’s homeroom teacher is Mr. Smith, in room 200.

 

If timetable changes result in a change of the student’s homeroom. You can easily change the homeroom on  the edit timetable page by selecting the More FAB then selecting the Edit homeroom icon shown here:

 

 

Selecting this icon will take you directly into the edit interface of the Grade info card, where grade, homeroom, advisor group and attendance type (full time, part time) may be edited.

 

Editing timetables for many students

Rather than editing students’ timetables one at a time through their student record, there are tools that will assist schools. For example, the scheduling functionality helps the school build their master timetable and then automatically loads the students into classes based on their course selections. Under the Student tools > Timetables option, there are tools that can automatically add or drop classes from a group of students, and tools to remove all classes in the timetable.

 

Show/hide unscheduled course selections

There are times when you may want to see the student’s course selections that are not scheduled into the student’s timetable. When the course selections are visible, the student’s preferred course selections that did not get into the student’s timetable will appear as courses at the bottom of the timetable grid. Here are a couple of situations where this feature is beneficial:

 

  • If the school transferred to the new school year without resolving all the conflicts, you can look at this year’s timetable along with the unscheduled course selections, thereby providing all the functionality to continue manually adjusting the student’s timetable in the current year.

  • During the scheduling process when working with the next year’s timetable, if the tool that automatically creates student timetables (load students) will not be run again, and there are conflicts that need to be resolved, one approach may be to look at the student’s next year’s timetable with the course selections visible. There are options in scheduling that go directly into edit next year's timetable with the unscheduled course selections showing automatically (under Scheduling next year > Resolve student conflicts > Students with conflicts and Individual student).

 

On the edit timetable tool bar, an icon will appear to indicate whether the timetable is showing or hiding unscheduled course selections. The tool tip will appear when you hover the icon, and you simply select the icon to change between showing or hiding the unscheduled course selections.

 

 

When the unscheduled course selections appear on the timetable grid, as shown below, you may proceed to edit the timetable as usual.

 

 

 

 

Note: only preferred course selections that have corresponding classes will appear. Alternate course selections (e.g. courses that may be substituted with preferred courses) will not appear.

 

Important note about course selections and next year’s timetable edits: Editing next year’s timetable is not recommended until the school is finished with any automated scheduling processes, specifically the load students process. When the classes in a student’s timetable are changed in such a manner that the student is now taking different courses than they originally requested when they first provided the school their course selections, it is important to know that the load students function will completely rebuild the student’s timetable based on the course selections only. For this reason, when a student’s next year’s timetable introduces course changes, remember to update the student's course selections if there is any possibility that the load students function will be run again.

 

Course fees

If a class is added to or deleted from a student’s timetable and the class has an associated Course Fee (as defined in the setup of the course), whenever timetable changes are made, the course fees are taken into consideration and refunds or course fees are automatically assigned (when appropriate). Please note, the ‘Refund after (days)’ calculation depends upon the start date of a class and the drop date within the student’s register and considers active days in the school calendar. If the class was dropped after the allotted refund period, the course fee will remain in the student's fees. For second semester classes, the count of number of days begins at the beginning of second semester. Use the Fees option in the student's record to confirm that fees are applied/refunded appropriately.

Printing Timetables

If a timetable needs to be printed, use the Print floating action button located in the lower right corner when viewing the student's timetable. Print is also available when editing the student's timetable, in the multi-FAB button, however, it will prompt to save the timetable or discard changes before it may be printed.

 

 

Use the OPTIONS and STUDENT INFORMATION tabs to specify printing parameters. For more info on these print options, see the Timetable option in Student reports.   Once the settings are correct, select the PRINT button in the bottom bar (below all the settings).
