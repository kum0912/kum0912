This option is used to create batch files, using the CREATE button. The batch file is uploaded to OnSIS through the OnSIS web site.  Once OnSIS processes the batch file, a results file is created. The results file is downloaded from OnSIS and is then processed using the PROCESS RESULTS button.  

 

This section describes the creation of the batch files. See the OnSIS Important concepts section for info on Updating data in OnSIS using the ‘Create/Upload/Process’ cycle.  

 

Choose OnSIS > Night/continuing education school.

 

 

Select the CREATE button on the ConEd/Nght school info card to see this form:

 

  

The OnSIS batch file options are detailed below. Prior to selecting the CREATE BATCH FILE option, determine whether other options on the form should be selected.   

 

Purge:  Y/N field. See Important concepts – Purge check boxes for more info.

 

Save results to CSV file (aka Also create CSV file): Y/N field. If set to Yes, the CREATE BATCH FILE option will create the requested batch file and the contents of the batch file will also appear in a comma delimited file (same file name as the XML batch file but with a CSV extension). Some boards/schools generate these csv files to perform data analysis on the information sent to OnSIS.

 

Purge and pre-populate all non-enrolled student info: Y/N field. If set to yes, it will include a command in the batch file that instructs OnSIS to purge and prepopulate all data for students who were submitted to OnSIS but these students did not actively attend school during this submission period.  This will purge and pre-populate diploma info, and option sheets for these non-enrolled students. It will also purge the SHSM program and certification info, and since this SHSM info is on the Student info card, it is important to remember to resubmit this file if you check this setting.

 

Adult day school – Exclude attendance totals: Y/N field (secondary schools only). Report card attendance data is reported to OnSIS for publicly funded secondary schools. If this school is considered an adult day school, then set this option to Y. If this is set to Yes, the following information is omitted from the OnSIS batch file: Total classes, Classes Missed, Times Late, IEP and ESL/ELD info.  

 

For classes with zero occupancy, check adds/drops to see if these classes were populated on the count date:  Y/N field (secondary schools only). Setting this to Yes would be the most accurate option, but it does significantly increase the time to create the file. If set to No, the file will automatically skip over any classes that have zero occupancy today (which may be after the count date). This makes the generation of the file faster, but should only be used if the school is certain that they have no empty classes that may be inadvertently omitted from the file.

 

Student class enrolment / other credits batch file

Default file name: S4_######_StuClsEnrol_yyyymmdd_hhmm.xml  

 

OnSIS is collecting enrolment information about the students enrolled in continuing education/night schools,  along with information regarding any credits attempted/earned during the academic school year.   Failed courses in the students' transcript are collected by OnSIS, along with withdrawn courses that are entered in the transcript as per full disclosure requirements. Any course not completed where the student did attend class, and therefore an ADE was generated will also be included in the OnSIS submission (if the ADE was generated within the submission period)

 

The setup of the school and the student’s timetables needs to be accurate for this batch file. Check the following:

 

  • Calendar start and end dates of each semester/term are accurate (Setup > School > Calendar).
  • Classes have accurate schedules (semester/term/day/period info) and start/end dates (Setup > Class info > Classes).
  • Students timetables are accurate (Student record > Class info > Timetable). The timetable should include all of the student’s classes. The date the student started each class, and if applicable, the date they withdrew from a class should be accurate. Hint: From the student’s record, us the Class info > Classes option to see the student’s start/end dates for each class.  If they need editing, from the timetable option, use the More icon to access the Register option (register of add/drop dates).
  • Students should have a graduation requirement assigned (Student record > Achievement > Graduation requirements). This is used to determine whether a completed course is compulsory for graduation.

 

The students' transcripts (completed courses) must be updated with the final marks and credits earned (see Add completed courses for info on this process). The info in the student’s completed courses (Student record > Achievement > Completed courses) is where the final marks and earned credit information is recorded for use in this file.

 

This batch file also includes ‘other credits’, an area within OnSIS that allows a school to report credits earned, when the course is not possible to be reported through the student class enrolment.   Other credits are those courses entered directly into the student’s completed courses (transcript) without adding the corresponding class into the student's timetable (e.g. conservatory music credits). Courses earned through the PLAR process are not considered other credits (they are reported in the PLAR batch file).

Prior Learning Assessment & Recognition (PLAR) credits

Default file name: S8_######_PLAR_yyyymmdd_hhmm.xml

 

 Prior Learning Assessment and Recognition (PLAR) is the formal evaluation and credit-granting process whereby students may obtain credits towards their secondary school diploma for prior learning.  PLAR applies to regular day school students and mature students who are enrolled in publicly funded secondary schools and inspected private schools that choose to implement PLAR. All boards must develop and implement policies and procedures related to the challenge and equivalency processes that are consistent with provincial policy.

 

The "challenge" process is the process whereby students' prior learning is assessed for the purpose of granting credit for a course developed from a provincial curriculum policy document.

 

The "equivalency" process is the process of assessing credentials from other jurisdictions.

 

All credits granted through the PLAR process are reported to OnSIS in the PLAR batch file. The courses in the student’s completed courses (transcript) that were earned during the submission period, that have the PLAR type field set to Challenge or Equivalency will be considered PLAR courses.

 

The fields collected by OnSIS are detailed in the Student info page (where regular schools need to report PLAR credits as well).

 

Prior Learning Assesment & Recognition (PLAR) credits batch file.

Specialist High Skills Major (SHSM) program & certifications  

 

Default file name: S15_######_SHSM_Program_yyyymmdd_hhmm.xml 

 

SHSM applies to publicly funded schools only.  

 

Specialist High Skills Major programs let students focus on a career path that matches their skills and interests while meeting the requirements of the Ontario Secondary School Diploma (OSSD), and they assist students in their transition from secondary school to apprenticeship training, college, university, or the workplace.  OnSIS collects info about students in the SHSM programs. The specific SHSM program is recorded in the student’s record (Achievement > Graduation requirements). Each program has unique sector-specific certificate requirements, and OnSIS collects these completed certifications as well as Sector Partnered Contextualized Experiences (SPCE) info. OnSIS will automatically determine whether a certificate is mandatory based on the student’s program, and if a student changes programs, the certificates earned in previous submissions will retain their original mandatory y/n designation. 

 

The fields collected by OnSIS in the SHSM program & certifications batch file are detailed below. Each SHSM program has specific course requirements and this info is reported in the SHSM courses batch file (found in the Other info option).  

 

The fields collected by OnSIS are detailed in the Student info page (where regular schools need to report SHSM info as well).

 

Specialist High Skills Major (SHSM) program & certifications batch file.
