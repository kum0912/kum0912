Scheduling next year > Resolve student conflicts - Overview

A student conflict occurs when a student is unable to get into all their preferred courses*. These students will need manual intervention by the school to adjust the student's timetable. This may involve adding one of the alternate courses to the student's timetable, or it may involve meeting with the student to pick alternate classes.  If the student was unable to get into more than 1 course, the student with multiple conflicts will have no classes in their timetable.  While editing the student's timetable, it is possible to see all their preferred courses as 'course selections' so that all the class sections are presented and the school can easily build a timetable with as many classes as possible, knowing which classes are most important for the student.

 

Typically student conflicts are not resolved until the school is done running load students. The load students process will rebuild the student's timetables based on the course selections, potentially undoing any manual conflict resolution done for the student (if the student's course selections are not updated to reflect the new courses). Resolving student conflicts may be done after transfer to current by editing the student's this year timetable (hint: use the icon to show preferred requests not yet placed in timetable).   

 

The Resolve student conflict options are used late in the scheduling process, specifically AFTER the school has completed these 2 processes for the very last time:

 

  • the classes are scheduled (the master timetable is complete) AND
  • the load students process has been run for the last time.

 

The tools to resolve student conflicts are as follows:

 

  1. Students with conflicts:  Use Scheduling next year > Resolve student conflicts > Students with conflicts to identify students who did not get assigned into all their preferred course selections*. Select a student to automatically open the student's next year timetable in edit mode. The course selections that were not able to be scheduled will appear for reference and the student's timetable may be adjusted as needed using the familiar student timetable functionality.  
  2. Individual student: Use Scheduling next year > Resolve student conflicts > Individual student to quickly access the edit next year's timetable option in a student's record. It will default to show the student's preferred requests not yet placed in timetable.
  3. Unbalanced timetables: Use Scheduling next year > Resolve student conflicts > Unbalanced timetables to see students with unbalanced timetables. The weights of the courses in each student's timetable are summed for each semester. If the total weight is different between semesters, this may be an indication of an unbalanced timetable with respect to the student's workload.

 

 

* If the load students tool substituted a preferred course with an alternate course, it is not considered a conflict.
